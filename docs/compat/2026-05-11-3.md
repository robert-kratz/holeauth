> **Notes**
> - *Framework middleware*: `@holeauth/nextjs-app-router` exports `createHoleauthMiddleware` via `./middleware`. Pages Router has no equivalent — Next.js middleware runs on the edge where the Pages Router Node.js bridge cannot execute.
> - *RBAC YAML on Hono*: `@holeauth/rbac-yaml` reads files via `node:fs` / `node:path`. Works on Node.js Hono deployments; blocked on Cloudflare Workers / Deno Deploy.
> - *tRPC Transparent Refresh*: `@holeauth/trpc` wires `getSessionOrRefresh` automatically for the App Router fetch adapter. Pages Router (`createHoleauthPagesContext`), Express (`createHoleauthExpressContext`), and Hono (`createHoleauthHonoContext`) expose session helpers but do not perform token rotation automatically.
> - *tRPC RBAC Guard*: `makePermissionProcedure` ships in `@holeauth/trpc`. For other frameworks, manual RBAC checks against `auth.rbac.*` APIs are required.
> - *Edge Runtime*: Pages Router bridges `IncomingMessage/ServerResponse` to the Web Fetch API at runtime — incompatible with edge. Express is Node.js-only. `@holeauth/core` uses a dynamic-import fallback for `@node-rs/argon2`; on edge, password hashing falls back to Web Crypto (bcrypt-equivalent).

## Roadmap Gaps

| Priority | Gap | Impact |
|---|---|---|
| 🔴 | No `@holeauth/nextjs-pages-router` middleware | Pages Router apps cannot intercept unauthenticated requests at the edge layer; every page/API route must call session helpers manually, which is error-prone and a common adoption blocker. |
| 🔴 | tRPC transparent refresh missing on Pages Router / Express / Hono | Three of the four supported frameworks require manual token-rotation logic in tRPC context factories. This creates an inconsistent DX and is a frequent source of silent auth failures after token expiry. |
| 🟡 | `makePermissionProcedure` / tRPC RBAC guard is App Router-only | Developers using Express or Hono with tRPC must write their own RBAC middleware; no official helper exists. |
| 🟡 | `@holeauth/rbac-yaml` blocked on Hono edge deployments (Cloudflare Workers / Deno Deploy) | Teams running Hono on serverless edge runtimes cannot use the YAML-driven RBAC config; they must wire groups programmatically via the DB adapter, losing the DX benefit of file-based config. |
| 🟢 | No dedicated non-Postgres IDP storage example / test | All five Drizzle adapters ship MySQL and SQLite sources but integration tests and documentation examples focus on Postgres; MySQL and SQLite production parity is unverified. |
