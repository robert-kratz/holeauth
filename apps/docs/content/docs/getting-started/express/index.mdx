# Express

This guide walks you through integrating holeauth into an Express application with Drizzle ORM. By the end, you will have email and password authentication, session management with automatic refresh-token rotation, and CSRF-protected endpoints.

### Create database schema

Holeauth provides a table factory for each Drizzle dialect. Import the factory that matches your database, pass your existing users table, and export the resulting tables for use in migrations.

<Tabs items={['Postgres', 'MySQL', 'SQLite', 'Headless']}>
  <Tab value="Postgres">
```ts title="db/schema.ts"
import { pgTable, text, timestamp } from 'drizzle-orm/pg-core';
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';

export const users = pgTable('user', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  email: text('email').notNull().unique(),
  emailVerified: timestamp('email_verified', { withTimezone: true, mode: 'date' }),
  passwordHash: text('password_hash'),
  name: text('name'),
  image: text('image'),
  createdAt: timestamp('created_at', { withTimezone: true, mode: 'date' }).notNull().defaultNow(),
});

export const holeauth = createHoleauthTables({ usersTable: users });
```
</Tab>

<Step>
### Create the auth instance

<Tabs items={['Postgres', 'MySQL', 'SQLite', 'Headless']}>
  <Tab value="Postgres">
```ts title="lib/auth.ts"
import { createExpressAuth } from '@holeauth/express';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '../db/client';
import { holeauth } from '../db/schema';

const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });

export const auth = createExpressAuth({
  secrets: {
    jwtSecret: process.env.HOLEAUTH_SECRET!,
  },
  adapters: {
    user: adapters.user,
    session: adapters.session,
    account: adapters.account,
    auditLog: adapters.auditLog,
    verificationToken: adapters.verificationToken,
    transaction: adapters.transaction,
  },
  tokens: {
    accessTtl: 60 * 15, // 15 minutes
    refreshTtl: 60 * 60 * 24 * 30, // 30 days
    cookiePrefix: 'holeauth',
    sameSite: 'lax',
  },
  registration: {
    selfServe: true,
    inviteTtlSeconds: 7 * 24 * 60 * 60, // 7 days
    inviteUrl: ({ token }) =>
      `${process.env.APP_URL}/register/accept?token=${encodeURIComponent(token)}`,
  },
  allowDangerousEmailAccountLinking: false,
});
```

<Step>
### Create the Express server

Build your Express server with cookie parsing enabled and mount the holeauth router:

```ts title="server.ts"
import express from 'express';
import cookieParser from 'cookie-parser';
import { auth } from './lib/auth';

const app = express();

app.use(express.json());
app.use(cookieParser());

// Mount the holeauth router at /api/auth
app.use('/api/auth', auth.router);

// Example protected route
app.get('/api/me', async (req, res) => {
  const { getSession } = await import('@holeauth/express');
  const session = await getSession(req, auth);
  
  if (!session) {
    return res.status(401).json({ error: 'Not authenticated' });
  }
  
  res.json({ user: session.user });
});

const port = process.env.PORT ?? 3000;
app.listen(port, () => {
  console.log(`Server listening on http://localhost:${port}`);
});
```

<Callout type="info">
  The `createExpressAuth` function returns an object containing both the holeauth instance API and an Express `.router` property. Mount the router with `app.use('/api/auth', auth.router)` to serve all auth endpoints.
</Callout>
</Step>

<Step>
### Read session in a protected route

Use the `getSession` helper to read the current session inside any Express route handler:

```ts title="routes/profile.ts"
import type { Request, Response } from 'express';
import { getSession } from '@holeauth/express';
import { auth } from '../lib/auth';

export async function getProfile(req: Request, res: Response) {
  const session = await getSession(req, auth);
  
  if (!session) {
    return res.status(401).json({ error: 'Not authenticated' });
  }
  
  // session is SessionData: { user: { id, email, name?, image? }, ... }
  return res.json({
    id: session.user.id,
    email: session.user.email,
    name: session.user.name ?? null,
    image: session.user.image ?? null,
  });
}
```

<Callout type="info">
  `getSession(req, auth)` reads the access-token cookie and validates the JWT. It returns `SessionData | null`. Use it to protect routes or conditionally serve content based on authentication state.
</Callout>
</Step>

## Configuration options

All configuration options for `createExpressAuth` are the same as [`defineHoleauth`](/packages/core#defineHoleauth) in `@holeauth/core`. The Express wrapper adds the `.router` property and handles cookie/request bridging automatically.

See [`@holeauth/core`](/packages/core) for the complete configuration reference, including token TTLs, registration policies, CSRF settings, and event hooks.
