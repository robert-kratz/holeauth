# Become an OIDC provider

This guide walks you through setting up your Hono application as an OAuth 2.0 / OpenID Connect Identity Provider. Other applications will be able to use your service as a single sign-on provider, similar to how you might integrate with Google or GitHub today.

<Callout type="info">
  This is the **provider** side — your app issues tokens to other applications. If you want to **consume** an external OIDC provider and sign users in via Google or GitHub, see [Sign in with Google, GitHub, and OIDC](/getting-started/hono/sso-consumer).
</Callout>

<Callout type="info">
  This guide assumes you have completed the [core Hono setup](/getting-started/hono).
</Callout>

<Tabs items={['Postgres', 'MySQL', 'SQLite']}>

```ts title="db/schema.ts" {3,13-14}
import { pgTable, text, timestamp } from 'drizzle-orm/pg-core';
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';
import { createIdpTables } from '@holeauth/idp-drizzle/pg';

export const users = pgTable('user', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  email: text('email').notNull().unique(),
  passwordHash: text('password_hash'),
  createdAt: timestamp('created_at', { withTimezone: true, mode: 'date' }).notNull().defaultNow(),
});

export const holeauth = createHoleauthTables({ usersTable: users });
export const idpSchema = createIdpTables();
```

<Step>
### Wire the plugin into your auth instance

<Tabs items={['Postgres', 'MySQL', 'SQLite']}>
  <Tab value="Postgres">
```ts title="lib/auth.ts" {4,5,9,23-27}
import { createHonoAuth } from '@holeauth/hono';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db/client';
import { holeauth, idpSchema } from '@/db/schema';
import { idp } from '@holeauth/plugin-idp';
import { createIdpAdapter } from '@holeauth/idp-drizzle/pg';

const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });
const idpAdapter = createIdpAdapter({ db, tables: idpSchema.tables });

export const auth = createHonoAuth({
  secrets: {
    jwtSecret: process.env.HOLEAUTH_SECRET!,
  },
  adapters: {
    user: adapters.user,
    session: adapters.session,
    account: adapters.account,
    auditLog: adapters.auditLog,
    verificationToken: adapters.verificationToken,
    transaction: adapters.transaction,
  },
  plugins: [
    idp({
      adapter: idpAdapter,
      issuer: process.env.APP_URL!,
    }),
  ],
  tokens: {
    accessTtl: 60 * 15,
    refreshTtl: 60 * 60 * 24 * 30,
    cookiePrefix: 'holeauth',
    sameSite: 'lax',
  },
  registration: {
    selfServe: true,
    inviteTtlSeconds: 7 * 24 * 60 * 60,
    inviteUrl: ({ token }) =>
      `${process.env.APP_URL}/register/accept?token=${encodeURIComponent(token)}`,
  },
  allowDangerousEmailAccountLinking: false,
});
```

<Step>
### New routes

The plugin automatically registers the following OIDC-compliant routes under `/api/auth`:

- `GET /.well-known/openid-configuration` — discovery document
- `GET /.well-known/jwks.json` — JSON Web Key Set
- `GET /oauth/authorize` — authorization endpoint
- `POST /oauth/token` — token endpoint
- `GET /oauth/userinfo` — user info endpoint
- `POST /oauth/revoke` — token revocation endpoint
- `POST /oauth/introspect` — token introspection endpoint
- `POST /oauth/clients` — register a new OAuth client (admin only)
- `GET /oauth/clients` — list all OAuth clients (admin only)

See [`@holeauth/plugin-idp`](/packages/plugin-idp) for detailed request and response schemas.
</Step>

<Step>
### Register an OAuth client

Before a third-party application can use your IDP, you must register it as an OAuth client:

```ts title="index.ts"
import { Hono } from 'hono';
import { auth } from './lib/auth';
import { getSession } from '@holeauth/hono';

const app = new Hono();

app.route('/api/auth', auth.app);

app.post('/api/oauth/clients', async (c) => {
  const session = await getSession(c, auth);
  if (!session) return c.json({ error: 'Unauthorized' }, 401);

  const body = await c.req.json();

  const client = await auth.idp.createClient({
    name: body.name,
    redirectUris: body.redirectUris,
    scopes: ['openid', 'profile', 'email'],
  });

  return c.json({
    clientId: client.clientId,
    clientSecret: client.clientSecret,
  });
});

export default app;
```

Store the `clientSecret` securely — it is only returned once.
</Step>
</Steps>

## How it works

When a third-party application initiates the OAuth flow:

1. The user is redirected to `GET /api/auth/oauth/authorize` with `client_id`, `redirect_uri`, and `state`.
2. Holeauth verifies the user is signed in. If not, redirects to `/login`.
3. The user consents (or consent is skipped if the client is pre-authorized).
4. Holeauth issues an authorization code and redirects back to the client's `redirect_uri`.
5. The client exchanges the code for an access token at `POST /api/auth/oauth/token`.
6. The client can now call `GET /api/auth/oauth/userinfo` with the access token to retrieve the user profile.
