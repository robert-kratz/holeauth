# Passkeys

This guide walks you through adding WebAuthn passkey authentication to your Hono application. Users will register a passkey using their device's biometric sensor or security key, then sign in without a password. Passkeys are the most secure and user-friendly authentication method available today.

<Callout type="info">
  This guide assumes you have completed the [core Hono setup](/getting-started/hono).
</Callout>

### Extend your database schema

Import the passkey table factory and add it to your schema:

<Tabs items={['Postgres', 'MySQL', 'SQLite']}>
  <Tab value="Postgres">
```ts title="db/schema.ts" {3,13-14}
import { pgTable, text, timestamp } from 'drizzle-orm/pg-core';
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';
import { createPasskeyTables } from '@holeauth/passkey-drizzle/pg';

export const users = pgTable('user', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  email: text('email').notNull().unique(),
  passwordHash: text('password_hash'),
  createdAt: timestamp('created_at', { withTimezone: true, mode: 'date' }).notNull().defaultNow(),
});

export const holeauth = createHoleauthTables({ usersTable: users });
export const passkeySchema = createPasskeyTables({ usersTable: users });
```

<Step>
### Wire the plugin into your auth instance

<Tabs items={['Postgres', 'MySQL', 'SQLite']}>
  <Tab value="Postgres">
```ts title="lib/auth.ts" {4,5,9,23-27}
import { createHonoAuth } from '@holeauth/hono';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db/client';
import { holeauth, passkeySchema } from '@/db/schema';
import { passkey } from '@holeauth/plugin-passkey';
import { createPasskeyAdapter } from '@holeauth/passkey-drizzle/pg';

const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });
const passkeyAdapter = createPasskeyAdapter({ db, tables: passkeySchema.tables });

export const auth = createHonoAuth({
  secrets: {
    jwtSecret: process.env.HOLEAUTH_SECRET!,
  },
  adapters: {
    user: adapters.user,
    session: adapters.session,
    account: adapters.account,
    auditLog: adapters.auditLog,
    verificationToken: adapters.verificationToken,
    transaction: adapters.transaction,
  },
  plugins: [
    passkey({
      adapter: passkeyAdapter,
      rpName: 'MyApp',
      rpId: process.env.APP_URL?.replace(/^https?:\/\//, '') ?? 'localhost',
    }),
  ],
  tokens: {
    accessTtl: 60 * 15,
    refreshTtl: 60 * 60 * 24 * 30,
    cookiePrefix: 'holeauth',
    sameSite: 'lax',
  },
  registration: {
    selfServe: true,
    inviteTtlSeconds: 7 * 24 * 60 * 60,
    inviteUrl: ({ token }) =>
      `${process.env.APP_URL}/register/accept?token=${encodeURIComponent(token)}`,
  },
  allowDangerousEmailAccountLinking: false,
});
```

  For production, set `rpId` to your root domain (e.g. `example.com`). Passkeys registered with one `rpId` cannot be used on a different one.

<Step>
### New routes

The plugin automatically registers the following routes under `/api/auth`:

- `POST /api/auth/passkey/register/begin` — starts passkey registration, returns a WebAuthn challenge
- `POST /api/auth/passkey/register/complete` — completes registration after the browser ceremony
- `POST /api/auth/passkey/login/begin` — starts passkey login, returns a WebAuthn challenge
- `POST /api/auth/passkey/login/complete` — completes login and issues session tokens
- `POST /api/auth/passkey/delete` — removes a passkey by credential ID

See [`@holeauth/plugin-passkey`](/packages/plugin-passkey) for detailed request and response schemas.
</Step>

<Step>
### Browser integration

Passkey registration and login require client-side JavaScript to invoke the WebAuthn API. The server routes return challenge objects that must be passed to `navigator.credentials.create()` (registration) or `navigator.credentials.get()` (login).

For a complete client-side implementation, see the [passkeys guide](/guides/passkeys).
</Step>
</Steps>

## Example: checking for existing passkeys

```ts title="index.ts"
import { Hono } from 'hono';
import { auth } from './lib/auth';
import { getSession } from '@holeauth/hono';

const app = new Hono();

app.route('/api/auth', auth.app);

app.get('/api/passkeys', async (c) => {
  const session = await getSession(c, auth);
  if (!session) return c.json({ error: 'Unauthorized' }, 401);

  const passkeys = await auth.passkey.listCredentials(session.user.id);

  return c.json({ passkeys });
});

export default app;
```
