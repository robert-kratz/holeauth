# Magic link and email OTP

Add passwordless sign-in to your Hono server. Users hit `POST /auth/magic-link/request` with an email and receive either a one-click magic link or a six-digit OTP.

<Callout type="info">
  This guide assumes you have completed the [core Hono setup](/getting-started/hono).
</Callout>

import { pgTable, text, timestamp } from 'drizzle-orm/pg-core';
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';

export const users = pgTable('user', {
  id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  email: text('email').notNull().unique(),
  emailVerified: timestamp('email_verified', { withTimezone: true, mode: 'date' }),
  name: text('name'),
  image: text('image'),

<Step>
### Wire the plugin into your auth instance

```ts title="lib/auth.ts" {5,6,11,30-44}
import { createHonoAuth } from '@holeauth/hono';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from './db/client';
import { holeauth, magicLinkSchema } from './db/schema';
import { magicLink } from '@holeauth/plugin-magic-link';
import { createMagicLinkAdapter } from '@holeauth/magic-link-drizzle/pg';
import { sendTransactionalEmail } from './email';

const AUTH_BASE_PATH = '/auth';
const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });
const magicLinkAdapter = createMagicLinkAdapter({ db, tables: magicLinkSchema.tables });

export const auth = createHonoAuth({
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  adapters: {
    user: adapters.user,
    session: adapters.session,
    account: adapters.account,
    auditLog: adapters.auditLog,
    verificationToken: adapters.verificationToken,
    transaction: adapters.transaction,
  },
  tokens: {
    accessTtl: 60 * 15,
    refreshTtl: 60 * 60 * 24 * 30,
    cookiePrefix: 'holeauth',
    sameSite: 'lax',
  },
  plugins: [
    magicLink({
      adapter: magicLinkAdapter,
      baseUrl: `${process.env.APP_URL}${AUTH_BASE_PATH}`,
      mode: 'both',
      role: 'primary',
      tokenTtlSeconds: 600,
      otpLength: 6,
      successRedirect: '/dashboard',
      errorRedirect: '/magic-link',
      expiredRedirect: '/magic-link/expired',
      sendEmail: async ({ email, url, code, type }) => {
        if (type === 'magic-link') await sendTransactionalEmail(email, 'Sign in', `Click: ${url}`);
        else await sendTransactionalEmail(email, 'Your code', `Code: ${code}`);
      },
    }),
  ] as const,
});
```

Mount the auth router as usual:

```ts title="server.ts"
import { Hono } from 'hono';
import { auth } from './lib/auth';

const app = new Hono();
app.route('/auth', auth.app);
export default app;
```

The plugin registers three new endpoints:

- `POST /auth/magic-link/request`
- `GET /auth/magic-link/consume?token=…`
- `POST /auth/magic-link/verify-otp`
</Step>

<Step>
### Call from your client

```ts
// request a magic link
await fetch('/auth/magic-link/request', {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify({ email, type: 'magic-link' }),
});

// or verify an OTP
await fetch('/auth/magic-link/verify-otp', {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify({ email, code }),
});
```

For the magic-link flow, `GET /auth/magic-link/consume?token=…` sets the session cookies and redirects to `successRedirect`.
</Step>
</Steps>

## Security notes

<Callout type="warn">
  Only the **hash** of each token / OTP is persisted. Use `https://` in production. Swap the default in-memory rate limiter for a Redis-backed one when scaling beyond a single process.
</Callout>
