# Roles and permissions

This guide walks you through adding role-based access control to your Next.js Pages Router project using `@holeauth/plugin-rbac` and `@holeauth/rbac-yaml`. You will define groups and permissions in a YAML file, assign users to groups, and enforce access control in API routes, `getServerSideProps`, and client components.

<Callout type="info">
  This guide assumes you have completed the [core Next.js Pages Router setup](/getting-started/nextjs-pages-router).
</Callout>

<Step>
### Define roles and permissions

Create a YAML file in your project root:

```yaml title="holeauth.rbac.yml"
groups:
  guest:
    default: true
    displayName: Guest
    description: Default group for all users
    priority: 0
    permissions:
      - posts.read

  member:
    displayName: Member
    description: Registered members with basic access
    priority: 10
    inherits:
      - guest
    permissions:
      - posts.create
      - posts.edit.own

  moderator:
    displayName: Moderator
    description: Moderators can edit and delete posts
    priority: 50
    inherits:
      - member
    permissions:
      - posts.edit.*
      - posts.delete.*

  admin:
    displayName: Administrator
    description: Full access to all resources
    priority: 100
    permissions:
      - '*'
```

Permission node syntax:

- `posts.read` — exact match
- `posts.*` — wildcard, matches `posts.create`, `posts.edit`, `posts.delete`, etc.
- `!posts.delete` — negation, explicitly denies a permission
- `*` — wildcard root, grants all permissions

Groups with `inherits` inherit all permissions from their parent groups. The `effective` permission list (after inheritance resolution) is what the plugin evaluates at runtime.

Exactly one group must have `default: true`. Users assigned no explicit groups receive the default group automatically.
</Step>

<Step>
### Wire the plugin into your auth instance

<Tabs items={['Postgres', 'MySQL', 'SQLite']}>
  <Tab value="Postgres">
```ts title="lib/auth.ts" {5,6,7,10,11,26-30,49}
import path from 'node:path';
import { createPagesAuthHandler } from '@holeauth/nextjs-pages-router';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db/client';
import { holeauth, rbacSchema } from '@/db/schema';
import { rbac } from '@holeauth/plugin-rbac';
import { createRbacAdapter } from '@holeauth/rbac-drizzle/pg';
import { loadRbacYaml } from '@holeauth/rbac-yaml';

const rbacYmlPath = path.join(process.cwd(), 'holeauth.rbac.yml');
const rbacYaml = loadRbacYaml(rbacYmlPath, { watch: process.env.NODE_ENV !== 'production' });

const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });
const rbacAdapter = createRbacAdapter({ db, tables: rbacSchema.tables });

export const auth = createPagesAuthHandler({
  secrets: {
    jwtSecret: process.env.HOLEAUTH_SECRET!,
  },
  adapters: {
    user: adapters.user,
    session: adapters.session,
    account: adapters.account,
    auditLog: adapters.auditLog,
    verificationToken: adapters.verificationToken,
    transaction: adapters.transaction,
  },
  plugins: [
    rbac({
      adapter: rbacAdapter,
      groups: rbacYaml.snapshot.groups,
    }),
  ],
  tokens: {
    accessTtl: 60 * 15,
    refreshTtl: 60 * 60 * 24 * 30,
    cookiePrefix: 'holeauth',
    sameSite: 'lax',
  },
  registration: {
    selfServe: true,
    inviteTtlSeconds: 7 * 24 * 60 * 60,
    inviteUrl: ({ token }) =>
      `${process.env.APP_URL}/register/accept?token=${encodeURIComponent(token)}`,
  },
  allowDangerousEmailAccountLinking: false,
});

rbacYaml.onReload((snapshot) => auth.rbac.reload(snapshot.groups));
```

<Step>
### Assign users to groups

Create an API route to assign groups:

```ts title="pages/api/rbac/assign-group.ts"
import type { NextApiRequest, NextApiResponse } from 'next';
import { getServerSidePropsSession } from '@holeauth/nextjs-pages-router';
import { auth } from '@/lib/auth';

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' });
  }

  const session = await getServerSidePropsSession({ req } as never, auth.config);
  if (!session) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  const canManageUsers = await auth.rbac.can(session.userId, 'users.manage');
  if (!canManageUsers) {
    return res.status(403).json({ error: 'Forbidden' });
  }

  const { userId, groupId } = req.body;
  await auth.rbac.assignGroup(userId, groupId);
  return res.status(200).json({ ok: true });
}
```
</Step>

<Step>
### Protect pages with getServerSideProps

Use the `withAuth` helper with permission guards:

```tsx title="pages/admin.tsx"
import { GetServerSideProps } from 'next';
import { withAuth } from '@holeauth/nextjs-pages-router';
import { auth } from '@/lib/auth';

export default function AdminPage() {
  return (
    <div>
      <h1>Admin dashboard</h1>
      <p>Only admins can see this page.</p>
    </div>
  );
}

export const getServerSideProps: GetServerSideProps = withAuth(
  async (ctx, session) => {
    return { props: {} };
  },
  auth.config,
  {
    permissions: ['admin.access'],
    redirectTo: '/login',
    forbiddenRedirect: '/forbidden',
  },
);
```

For more complex permission checks, use the RBAC API directly:

```tsx title="pages/posts/[id]/edit.tsx"
import { GetServerSideProps } from 'next';
import { getServerSidePropsSession } from '@holeauth/nextjs-pages-router';
import { auth } from '@/lib/auth';

interface EditPostProps {
  postId: string;
  canEdit: boolean;
}

export default function EditPostPage({ postId, canEdit }: EditPostProps) {
  if (!canEdit) {
    return <div>You do not have permission to edit this post.</div>;
  }

  return (
    <div>
      <h1>Edit post {postId}</h1>
    </div>
  );
}

export const getServerSideProps: GetServerSideProps<EditPostProps> = async (ctx) => {
  const session = await getServerSidePropsSession(ctx, auth.config);
  if (!session) {
    return { redirect: { destination: '/login', permanent: false } };
  }

  const postId = ctx.params?.id as string;
  const canEditAny = await auth.rbac.can(session.userId, 'posts.edit.*');
  const canEditOwn = await auth.rbac.can(session.userId, 'posts.edit.own');

  return {
    props: {
      postId,
      canEdit: canEditAny || canEditOwn,
    },
  };
};
```
</Step>

<Step>
### Use RBAC hooks in client components

The `useRbac` hook provides reactive permission checks:

```tsx title="components/post-actions.tsx"
'use client';
import { useRbac } from '@holeauth/react';

export function PostActions({ postId }: { postId: string }) {
  const { can } = useRbac();

  const canEdit = can('posts.edit.*');
  const canDelete = can('posts.delete.*');

  return (
    <div>
      {canEdit && <button>Edit</button>}
      {canDelete && <button>Delete</button>}
    </div>
  );
}
```
</Step>

## How it works

The RBAC plugin resolves permissions in layers:

1. User is assigned to one or more groups via the `user_groups` table.
2. Each group has a list of permissions (wildcards and negations supported).
3. Groups can inherit from other groups, creating a priority chain.
4. The effective permission set is computed once per session and embedded in the access token.
5. Permission checks use wildcard matching: `posts.*` matches `posts.create`, `posts.edit`, etc.

<Callout type="warn">
  Permission changes do not take effect until the user refreshes their access token. For immediate enforcement, revoke all sessions for the user.
</Callout>
