# Adapters

holeauth never touches your database directly. Instead, it defines a set of adapter interfaces that abstract persistence operations. You provide an adapter implementation, and holeauth calls its methods to create, read, update, and delete users, sessions, and audit logs.

This design keeps the core framework-agnostic and lets you use any ORM or database client.

## Adapter Interfaces

Every holeauth instance requires four adapters:

<TypeTable
  type={{
    user: {
      description: 'Create, read, update, and delete user accounts.',
      type: 'UserAdapter',
      required: true,
    },
    session: {
      description: 'Create, read, rotate, and revoke sessions.',
      type: 'SessionAdapter',
      required: true,
    },
    auditLog: {
      description: 'Persist and optionally query audit events.',
      type: 'AuditLogAdapter',
      required: true,
    },
    account: {
      description: 'Link and unlink OAuth/OIDC accounts (for SSO).',
      type: 'AccountAdapter',
      required: false,
    },
    verificationToken: {
      description: 'Store and consume short-lived tokens (password reset, email verification).',
      type: 'VerificationTokenAdapter',
      required: false,
    },
    transaction: {
      description: 'Wrap multi-step writes in a database transaction.',
      type: 'TransactionAdapter',
      required: false,
    },
  }}
/>

The `user`, `session`, and `auditLog` adapters are mandatory. The others are optional and only required if you use the corresponding features (SSO, password reset, transactions).

## UserAdapter

  type={{

      type: '(email: string) => Promise<AdapterUser | null>',

    },
    createUser: {
      description: 'Insert a new user row.',
      type: '(data: Omit<AdapterUser, "id">) => Promise<AdapterUser>',

      type: '(id: string, patch: Partial<AdapterUser>) => Promise<AdapterUser>',

      type: '(id: string) => Promise<void>',

The `AdapterUser` shape:

<TypeTable
  type={{
    id: {
      description: 'Unique user ID (UUID or auto-increment).',
      type: 'string',
      required: true,
    },
    email: {
      description: 'User email address (unique).',
      type: 'string',
      required: true,
    },
    emailVerified: {
      description: 'Timestamp when email was verified.',
      type: 'Date | null',
      required: false,
    },
    name: {
      description: 'User display name.',
      type: 'string | null',
      required: false,
    },
    image: {
      description: 'Avatar URL.',
      type: 'string | null',
      required: false,
    },
    passwordHash: {
      description: 'Argon2id or PBKDF2 hash of the user password.',
      type: 'string | null',
      required: false,
    },
  }}
/>

<Callout type="warn">
  Never store plain-text passwords in `passwordHash`. Always hash them using `password.hash` from `@holeauth/core/password`.
</Callout>

      type: 'string',

    expiresAt: {
      description: 'Expiration timestamp of the refresh token.',
      type: 'Date',
      required: true,
    },
    createdAt: {
      description: 'Timestamp when the session was created.',
      type: 'Date',
      required: false,
    },
    revokedAt: {
      description: 'Timestamp when the session was revoked (null if active).',
      type: 'Date | null',
      required: false,
    },
    ip: {
      description: 'IP address of the client that created the session.',
      type: 'string | null',
      required: false,
    },
    userAgent: {
      description: 'User-Agent header of the client that created the session.',
      type: 'string | null',
      required: false,
    },
  }}
/>

<Callout type="info">
  The `rotateRefresh` method must be atomic. Use a database transaction or an UPDATE statement that modifies both `refreshTokenHash` and `expiresAt` in a single query.
</Callout>

## AuditLogAdapter

<TypeTable
  type={{
    record: {
      description: 'Persist an audit event. Must be awaited by every flow.',
      type: '(event: AdapterAuditEvent) => Promise<void>',
      required: true,
    },
    list: {
      description: 'Query audit events by user, type, or timestamp.',
      type: '(filter: { userId?: string; type?: string; limit?: number }) => Promise<AdapterAuditEvent[]>',
      required: false,
    },
  }}
/>

The `record` method is mandatory. The `list` method is optional and used for displaying audit logs in a dashboard.

## TransactionAdapter

Optional. When provided, holeauth wraps multi-step writes in a transaction:

<TypeTable
  type={{
    run: {
      description: 'Execute a function inside a database transaction.',
      type: '<T>(fn: () => Promise<T>) => Promise<T>',
      required: true,
    },
  }}
/>

Implementations should propagate the transaction context to all adapter method calls inside `fn`. Use async-local-storage or return a fresh adapter bundle bound to the transaction.

## Drizzle Adapter

The `@holeauth/adapter-drizzle` package provides reference implementations for Postgres, MySQL, and SQLite. It exports factory functions under subpath exports:

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { drizzleAdapter } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
});
```

See the [Drizzle Adapter](/packages/adapter-drizzle) page for details on how to generate schema and wire the adapter.

## Headless Implementation

If you use a different ORM (Prisma, Kysely, TypeORM), implement the adapter interfaces yourself:

```ts title="lib/adapters/user.ts"
import type { UserAdapter, AdapterUser } from '@holeauth/core/adapters';
import { prisma } from '@/db';

export const userAdapter: UserAdapter = {
  async getUserById(id: string) {
    return await prisma.user.findUnique({ where: { id } });
  },
  async getUserByEmail(email: string) {
    return await prisma.user.findUnique({ where: { email } });
  },
  async createUser(data: Omit<AdapterUser, 'id'>) {
    return await prisma.user.create({ data });
  },
  async updateUser(id: string, patch: Partial<AdapterUser>) {
    return await prisma.user.update({ where: { id }, data: patch });
  },
  async deleteUser(id: string) {
    await prisma.user.delete({ where: { id } });
  },
};
```

Then wire it into the config:

```ts title="lib/auth.ts"
export const auth = defineHoleauth({
  adapters: {
    user: userAdapter,
    session: sessionAdapter,
    auditLog: auditLogAdapter,
  },
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
});
```
