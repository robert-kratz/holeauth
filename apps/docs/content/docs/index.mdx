# holeauth

holeauth is a modular, edge-native authentication ecosystem for modern TypeScript applications. It provides production-grade auth primitives behind a clean adapter pattern—you bring your database, holeauth brings cryptography, token lifecycle management, and framework integration.

The core is framework-agnostic with runtime-detected primitives that work on Node.js, Bun, and edge runtimes. First-class Next.js integration is available today with route handlers and middleware for the App Router.

Every authentication flow—from password sign-in to WebAuthn ceremonies—is built on a composable plugin architecture. Enable only the features you need: two-factor authentication, passkeys, role-based access control, SSO consumer, or even become an OpenID Connect identity provider yourself.

## Security Model

holeauth is built with security at its foundation:

- Rotating refresh tokens with family tracking and automatic reuse detection
- Double-submit CSRF protection for all mutating endpoints
- Argon2id password hashing with runtime fallback to PBKDF2 on edge
- Constant-time string comparisons for tokens and passwords
- Mandatory audit logging for every authentication event
- HttpOnly cookies for access and refresh tokens

All cryptographic operations use Web Crypto API primitives and the battle-tested jose library for JWT signing and verification.
