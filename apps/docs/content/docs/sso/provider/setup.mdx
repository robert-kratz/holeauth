    ### Create the IdP schema

    The plugin stores OAuth apps, authorization codes, refresh tokens, consent records, signing keys, and teams in dedicated tables.

    <Tabs items={['PostgreSQL', 'MySQL', 'SQLite']}>
      <Tab value="PostgreSQL">
```ts title="db/schema.ts"
import { pgTable } from 'drizzle-orm/pg-core';
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';
import { createIdpTables } from '@holeauth/idp-drizzle/pg';

export const holeauth = createHoleauthTables(pgTable);
export const idp = createIdpTables(pgTable);
```
      </Tab>
      <Tab value="MySQL">

  <Step>
    ### Wire the plugin into holeauth

    ```ts title="lib/auth.ts" {4-5,12-21}
import { defineHoleauth } from '@holeauth/core';
import { drizzleAdapter } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db';
import { idp } from '@holeauth/plugin-idp';
import { drizzleIdpAdapter } from '@holeauth/idp-drizzle/pg';
import * as schema from '@/db/schema';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db, schema.holeauth),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  plugins: [
    idp({
      adapter: drizzleIdpAdapter(db, schema.idp),
      issuer: process.env.OIDC_ISSUER!,
      scopesSupported: ['openid', 'profile', 'email', 'offline_access'],
      accessTokenTtl: 900,
      idTokenTtl: 900,
      refreshTokenTtl: 60 * 60 * 24 * 30,
      authorizationCodeTtl: 600,
      signingAlg: 'RS256',
    }),
  ],
});
```

  <Step>
    ### Register an OAuth application

    Before clients can authenticate, you must register an OAuth app. Use the plugin API from a server action, admin script, or seed file:

    ```ts title="scripts/seed-oauth-app.ts"
import { auth } from '@/lib/auth';

const idpPlugin = auth.getPlugin('idp');

const { app, clientSecret } = await idpPlugin.apps.create(
  'admin-user-id',
  {
    name: 'My Mobile App',
    description: 'iOS and Android clients',
    type: 'public',
    redirectUris: ['myapp://oauth/callback'],
    allowedScopes: ['openid', 'profile', 'email', 'offline_access'],
    requirePkce: true,
  },
);

console.log('Client ID:', app.id);
if (clientSecret) {
  console.log('Client Secret:', clientSecret);
  console.log('⚠️  Save this secret now — it cannot be retrieved later.');
}
```

    <Callout type="warn">
      For `type: "confidential"` apps, the returned `clientSecret` is shown **only once**. It is hashed (SHA-256) before storage and cannot be retrieved later. Store it securely or regenerate via `idpPlugin.apps.regenerateSecret(userId, appId)`.
    </Callout>

    **App types:**
    - `"public"` — Single-page apps, mobile apps. No client secret. PKCE required.
    - `"confidential"` — Server-to-server, traditional web apps. Client secret required for token endpoint.
  </Step>

  <Step>
    ### Verify discovery endpoint

    Confirm that the OIDC metadata is exposed:

    ```bash
    curl https://auth.example.com/.well-known/openid-configuration
    ```

    Expected response (abbreviated):

    ```json
    {
      "issuer": "https://auth.example.com",
      "authorization_endpoint": "https://auth.example.com/oauth2/authorize",
      "token_endpoint": "https://auth.example.com/oauth2/token",
      "userinfo_endpoint": "https://auth.example.com/oauth2/userinfo",
      "jwks_uri": "https://auth.example.com/oauth2/jwks",
      "revocation_endpoint": "https://auth.example.com/oauth2/revoke",
      "end_session_endpoint": "https://auth.example.com/oauth2/end-session",
      "response_types_supported": ["code"],
      "grant_types_supported": ["authorization_code", "refresh_token"],
      "code_challenge_methods_supported": ["S256"],
      "scopes_supported": ["openid", "profile", "email", "offline_access"],
      "id_token_signing_alg_values_supported": ["RS256"]
    }
    ```
