# Sign in with GitHub

GitHub is an **OAuth 2.0** provider (not OIDC). holeauth fetches the user profile from GitHub's REST API (`https://api.github.com/user`) and normalizes it into the same shape as OIDC providers.

<Callout type="info">
  GitHub does not issue ID tokens. holeauth uses the access token to fetch user data from `/user` and (if the `user:email` scope is granted) `/user/emails` to get the primary email address.
</Callout>

<Steps>
  <Step>
    ### Create a GitHub OAuth App

    1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
    2. Click **New OAuth App**
    3. Fill in:
       - **Application name**: Your app name
       - **Homepage URL**: `http://localhost:3000` (or your production URL)
       - **Authorization callback URL**: `http://localhost:3000/api/auth/sso/github/callback`
    4. Click **Register application**
    5. Copy the **Client ID**
    6. Click **Generate a new client secret** and copy it
  </Step>

  <Step>
    ### Set environment variables

    ```bash title=".env.local"
    GITHUB_CLIENT_ID=Iv1.abc...
    GITHUB_CLIENT_SECRET=abc123...
    ```
  </Step>

  <Step>
    ### Wire `GithubProvider` into holeauth

    ```ts title="lib/auth.ts" {4,10-15}
import { defineHoleauth } from '@holeauth/core';
import { drizzleAdapter } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db';
import { GithubProvider } from '@holeauth/core/sso';
import * as schema from '@/db/schema';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db, schema.holeauth),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  providers: [
    GithubProvider({
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
      redirectUri: 'http://localhost:3000/api/auth/sso/github/callback',
    }),
  ],
});
```

  <Step>
    ### Add a sign-in button

    ```tsx title="app/login/page.tsx"
export default function LoginPage() {
  return (
    <div>
      <h1>Sign in</h1>
      <a href="/api/auth/sso/github/authorize">
        <button>Sign in with GitHub</button>
      </a>
    </div>
  );
}
```

    When clicked, the user is redirected to GitHub's OAuth consent screen. After approval, GitHub redirects back to `/api/auth/sso/github/callback`, where holeauth:
    1. Exchanges the authorization code for an access token
    2. Fetches user profile from `https://api.github.com/user`
    3. Fetches primary email from `https://api.github.com/user/emails` (if `user:email` scope granted)
    4. Creates a new user or links the GitHub account to an existing user
    5. Issues a holeauth session
  </Step>

  <Step>
    ### Handle the result

    ```tsx title="app/dashboard/page.tsx"
import { auth } from '@/lib/auth';
import { validateCurrentRequest } from '@holeauth/nextjs/server';

export default async function DashboardPage() {
  const session = await validateCurrentRequest(auth);
  if (!session) return <div>Not signed in</div>;

  return <div>Welcome, {session.user.email}!</div>;
}
```
  </Step>
</Steps>

## Email address handling

GitHub users can have **multiple email addresses** and may choose not to make their primary email public. holeauth:
1. Fetches the user profile from `/user` (includes `email` if public)
2. If `email` is null, fetches `/user/emails` and selects the primary verified email
3. Throws `ProviderError` if no email is found

<Callout type="warn">
  If a GitHub user has not verified their primary email or has hidden their email from public view, holeauth will fetch it from `/user/emails` (requires the `user:email` scope). Ensure the `user:email` scope is granted in your OAuth app configuration.
</Callout>

## Account linking

If a user signs in with GitHub and an existing holeauth user has the same email address:
- By default, holeauth throws `AccountConflictError` (safe mode).
- If `allowDangerousEmailAccountLinking: true` is set **and** GitHub verified the email, holeauth auto-links the accounts.

<Callout type="warn">
  GitHub **does** verify email addresses (the primary email is always verified). Auto-linking is safe if you trust GitHub as a provider. However, GitHub users can change their email address without re-verification, so consider the risk.
</Callout>

## Tokens stored

GitHub returns:
- `access_token` — Bearer token for GitHub API calls (stored in `holeauth_accounts.access_token`)
- `scope` — Granted scopes (stored in `holeauth_accounts.scope`)

GitHub does **not** issue refresh tokens for OAuth Apps. If you need refresh tokens, use a [GitHub App](https://docs.github.com/en/apps/creating-github-apps/about-creating-github-apps/about-creating-github-apps) instead.
