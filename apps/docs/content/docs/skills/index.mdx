# AI agent integration guide

Holeauth ships **skills** — self-contained `SKILL.md` files that teach an AI coding agent (Claude, Copilot, Cursor, etc.) exactly how to [integrate](/getting-started) or [extend](/plugins) holeauth without hallucinating the API.

Each skill is:

- **Trigger-driven** — front-matter description tells the agent when to invoke it automatically
- **Interactive** — uses `vscode_askQuestions` (or equivalent) to interview you before generating code
- **Context-frugal** — embeds only hardcoded gotchas; everything else is fetched on demand from the [Search API](/skills/search-api)

## Setup

<Steps>
<Step>

### Install the custom instructions file

The instructions file gives your agent persistent context about holeauth: which packages exist, where the docs live, and key runtime constraints (e.g. Edge-safe middleware). It applies to every file in your project.

```
https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md
```

<Tabs items={['GitHub Copilot', 'Claude Code', 'Cursor', 'Manual']}>
  <Tab value="GitHub Copilot">
    Save the file to `.github/instructions/holeauth.instructions.md` in your repo. VS Code Copilot auto-discovers instruction files in that directory and applies them globally.

    ```bash
    curl -fsSL https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md \
      -o .github/instructions/holeauth.instructions.md
    ```
  </Tab>
  <Tab value="Claude Code">
    Paste the raw URL into your `AGENTS.md` under a `## Instructions` section:

    ```md title="AGENTS.md"
    ## Instructions
    - [holeauth context](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md)
    ```
  </Tab>
  <Tab value="Cursor">
    Add the raw URL to your `.cursorrules`, or paste the file contents directly into a project rule in Cursor settings.
  </Tab>
  <Tab value="Manual">
    Open the raw URL, copy the contents, and paste them into your agent's system prompt or project-level instructions.
  </Tab>
</Tabs>

</Step>
<Step>

### Install skills

Skills are scoped to specific tasks. Install the ones you need — start with `integrate-holeauth` for existing projects or `bootstrap-nextjs-holeauth` for greenfield setups.

Every skill lives at a stable raw URL:

```
https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/<skill-name>/SKILL.md
```

<Tabs items={['GitHub Copilot', 'Claude Code', 'Cursor', 'Manual']}>
  <Tab value="GitHub Copilot">
    Drop each `SKILL.md` into `.github/skills/<name>/SKILL.md` in your repo. VS Code Copilot auto-discovers and registers them via the front-matter `description` field.

    ```bash
    # Example: install the core integration skill
    mkdir -p .github/skills/integrate-holeauth
    curl -fsSL https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/integrate-holeauth/SKILL.md \
      -o .github/skills/integrate-holeauth/SKILL.md
    ```
  </Tab>
  <Tab value="Claude Code">
    Add skill URLs to your project's `AGENTS.md`:

    ```md title="AGENTS.md"
    ## Skills
    - [bootstrap-nextjs-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/bootstrap-nextjs-holeauth/SKILL.md)
    - [integrate-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/integrate-holeauth/SKILL.md)
    ```
  </Tab>
  <Tab value="Cursor">
    Reference the raw URLs from `.cursorrules` or paste the skill contents into a project rule.
  </Tab>
  <Tab value="Manual">
    Open the raw URL for each skill you need, copy the contents, and feed them to your agent as a system instruction.
  </Tab>
</Tabs>

<Callout type="info">
  Skills are versioned alongside holeauth packages. Pin to a release tag (`/tree/v1.2.0/`) for reproducible scaffolds, or track `main` for the latest API surface.
</Callout>

</Step>
<Step>

### Bootstrap with the agent prompt

Paste this prompt into your AI agent to interactively pick and install skills **and** the instructions file in one shot. The agent will ask which skills you want, then fetch and write all files automatically.

```text title="Copy → paste into your AI agent"
Browse https://github.com/robert-kratz/holeauth/tree/main/.github/skills to list available skills. Use vscode_askQuestions to ask: (1, multiSelect) which skills to install; (2, boolean) whether to install the holeauth context instructions file. For each selected skill: fetch https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/<name>/SKILL.md → write to .github/skills/<name>/SKILL.md. If installing instructions: fetch https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md → write to .github/instructions/holeauth.instructions.md. For any file that already exists, overwrite only if remote content changed (report updates).
```

</Step>
</Steps>
