Holeauth ships **skills** — self-contained `SKILL.md` files that teach an AI coding agent (Claude, Copilot, Cursor, etc.) exactly how to integrate or extend holeauth without hallucinating the API.

Each skill is:

- **Trigger-driven** — front-matter description tells the agent when to invoke it
- **Interactive** — uses `vscode_askQuestions` (or equivalent) to interview the user before generating code
- **Context-frugal** — embeds only hardcoded gotchas; everything else is fetched on demand from this docs site

## Search API

Skills that need deeper detail call the docs search API rather than ballooning their own context:

```
GET https://docs.holeauth.dev/api/search?q=<term>
```

Example queries an agent might issue:

- `?q=rbac+yaml+wildcards`
- `?q=passkey+rpID+origin`
- `?q=plugin+definePlugin+hooks`
- `?q=middleware+protectAllExcept`

The response is a flat list of `{ id, url, content, type }` records covering every page on this site.

## Install a skill

Every skill lives at a stable raw URL on GitHub:

```
https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/<skill-name>/SKILL.md
```

<Tabs items={['Claude Code', 'GitHub Copilot', 'Cursor', 'Manual']}>
  <Tab value="Claude Code">
    Add to your project's `AGENTS.md`:

    ```md
    ## Skills
    - [bootstrap-nextjs-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/bootstrap-nextjs-holeauth/SKILL.md)
    - [integrate-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/integrate-holeauth/SKILL.md)
    ```
  </Tab>
  <Tab value="GitHub Copilot">
    Drop each `SKILL.md` into `.github/skills/<name>/SKILL.md` in your repo. VS Code Copilot auto-discovers them via the front-matter `description`.
  </Tab>
  <Tab value="Cursor">
    Reference the raw URLs from `.cursorrules` or paste the contents into a project rule.
  </Tab>
  <Tab value="Manual">
    Open the raw URL, copy the contents, and feed it to your agent as a system instruction.
  </Tab>
</Tabs>

<Callout type="info">
  Skills are versioned alongside the rest of holeauth. Pin to a tag (`/tree/v1.2.0/`) for reproducible scaffolds, or track `main` for the latest API surface.
</Callout>

## How a skill is structured

Every `SKILL.md` follows the same shape so agents can parse them deterministically:

```md title=".github/skills/<name>/SKILL.md"
---
name: <kebab-case-name>
description: "<comprehensive trigger phrases — first line that VS Code/Copilot uses>"
argument-hint: "<optional hint>"
---

# <Title>

<one-sentence purpose>

## Prerequisites
## Source of truth
## Procedure
### Step 1 — Interview (vscode_askQuestions table)
### Step 2 — Install
### Step 3+ — Implementation
## Hardcoded gotchas
## Need more detail?
GET https://docs.holeauth.dev/api/search?q=<topic>
```
