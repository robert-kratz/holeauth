# Skills for AI Agents

Holeauth ships **skills** — self-contained `SKILL.md` files that teach an AI coding agent (Claude, Copilot, Cursor, etc.) exactly how to [integrate](/getting-started) or [extend](/plugins) holeauth without hallucinating the API.

Each skill is:

- **Trigger-driven** — front-matter description tells the agent when to invoke it
- **Interactive** — uses `vscode_askQuestions` (or equivalent) to interview the user before generating code
- **Context-frugal** — embeds only hardcoded gotchas; everything else is fetched on demand from this docs site via the [Search API](#search-api)

## Install a skill

Every skill lives at a stable raw URL on GitHub:

```
https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/<skill-name>/SKILL.md
```

<Tabs items={['Claude Code', 'GitHub Copilot', 'Cursor', 'Manual']}>
  <Tab value="Claude Code">
    Add to your project's `AGENTS.md`:

    ```md
    ## Skills
    - [bootstrap-nextjs-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/bootstrap-nextjs-holeauth/SKILL.md)
    - [integrate-holeauth](https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/integrate-holeauth/SKILL.md)
    ```
  </Tab>
  <Tab value="GitHub Copilot">
    Drop each `SKILL.md` into `.github/skills/<name>/SKILL.md` in your repo. VS Code Copilot auto-discovers them via the front-matter `description`.
  </Tab>
  <Tab value="Cursor">
    Reference the raw URLs from `.cursorrules` or paste the contents into a project rule.
  </Tab>
  <Tab value="Manual">
    Open the raw URL, copy the contents, and feed it to your agent as a system instruction.
  </Tab>
</Tabs>

<Callout type="info">
  Skills are versioned alongside the rest of holeauth. Pin to a tag (`/tree/v1.2.0/`) for reproducible scaffolds, or track `main` for the latest API surface.
</Callout>

### Custom instructions file

For a persistent holeauth context that applies to every file in your project, install the custom instruction file. It tells your AI agent where the docs live, which packages exist, and enforces key runtime rules (e.g. Edge-safe middleware).

```
https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md
```

Save it to `.github/instructions/holeauth.instructions.md` in your project. VS Code Copilot and compatible agents auto-discover files in that directory.

### Install via agent prompt

Paste this prompt into your AI agent to pick and install skills **and** the custom instructions file interactively:

```text title="Install holeauth skills + instructions (copy → agent)"
Browse https://github.com/robert-kratz/holeauth/tree/main/.github/skills to list available skills. Use vscode_askQuestions to ask: (1, multiSelect) which skills to install; (2, boolean) whether to install the holeauth context instructions file. For each selected skill: fetch https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/skills/<name>/SKILL.md → write to .github/skills/<name>/SKILL.md. If installing instructions: fetch https://raw.githubusercontent.com/robert-kratz/holeauth/main/.github/instructions/holeauth.instructions.md → write to .github/instructions/holeauth.instructions.md. For any file that already exists, overwrite only if remote content changed (report updates).
```

## Search API

Skills that need deeper detail call the docs search API rather than ballooning their own context:

```
GET https://docs.holeauth.dev/api/search?q=<term>
```

Example queries an agent might issue:

- `?q=rbac+yaml+wildcards` → [RBAC docs](/packages/plugin-rbac)
- `?q=passkey+rpID+origin` → [Passkey docs](/packages/plugin-passkey)
- `?q=plugin+definePlugin+hooks` → [Plugin architecture](/plugins/define-plugin)
- `?q=middleware+protectAllExcept` → [Next.js App Router](/getting-started/nextjs-app-router)

The response is a flat list of `{ id, url, content, type }` records covering every page on this site.

## How a skill is structured

Every `SKILL.md` follows the same shape so agents can parse them deterministically:

```md title=".github/skills/<name>/SKILL.md"
---
name: <kebab-case-name>
description: "<comprehensive trigger phrases — first line that VS Code/Copilot uses>"
argument-hint: "<optional hint>"
---

# <Title>

<one-sentence purpose>

## Prerequisites
## Source of truth
## Procedure
### Step 1 — Interview (vscode_askQuestions table)
### Step 2 — Install
### Step 3+ — Implementation
## Hardcoded gotchas
## Need more detail?
GET https://docs.holeauth.dev/api/search?q=<topic>
```

The `## Prerequisites` section references the [core setup skill](/skills) as a prerequisite for all plugin skills. The `## Hardcoded gotchas` section documents known pitfalls such as [Edge runtime constraints](/getting-started/nextjs-app-router) and [peer dependency mismatches](/packages/core).

<Callout type="info">
  Want to contribute a skill? Open a PR adding a folder under `.github/skills/<name>/SKILL.md` and a row in the registry above. See the [plugin architecture docs](/plugins/define-plugin) for how skills map to the plugin API.
</Callout>
