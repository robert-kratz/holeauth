# Email & Password

The simplest holeauth flow: register a user, sign them in, and let them sign out.

## What you need

- `adapters.user`
- `adapters.session`
- `adapters.auditLog`

Passwords are hashed with Argon2id (if `argon2` is installed on Node) or PBKDF2-SHA256 otherwise. See [`@holeauth/core` → password](/docs/packages/core#password).

## Register

`POST /api/auth/register`

```ts
await fetch('/api/auth/register', {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  body: JSON.stringify({
    email: 'ada@example.com',
    password: 'correct-horse-battery-staple',
    name: 'Ada Lovelace', // optional
  }),
});
```

- Returns `409 ACCOUNT_CONFLICT` if the email is already in use.
- Does **not** automatically sign the user in — call `/signin` next if you want auto login.

Under the hood:

1. `adapters.user.getUserByEmail(email)` — must return `null`.
2. Password is hashed via `password.hash`.
3. `adapters.user.createUser({ email, passwordHash, name })` is called.
4. A `user.registered` event is emitted.

## Sign in

`POST /api/auth/signin`

```ts
const res = await fetch('/api/auth/signin', {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  body: JSON.stringify({ email, password }),
});

const json = await res.json();
if (json.pending2fa) {
  // TOTP required — see the Two-Factor guide
  window.location.href = '/2fa/verify';
} else {
  window.location.href = '/dashboard';
}
```

Responses:

- `{ ok: true, user, expiresAt }` — access, refresh and CSRF cookies are set.
- `{ pending2fa: true, pendingExpiresAt }` — a short-lived pending cookie is set; the client must call `/2fa/verify` within `pending2faTtl` (default 5 minutes).
- `401 INVALID_CREDENTIALS` — email unknown or password wrong. Same error for both: never leak which one.

## Sign out

`POST /api/auth/signout` — **requires CSRF**.

```ts
// read the CSRF cookie, echo it in the header
const csrf = document.cookie
  .split('; ')
  .find((c) => c.startsWith('holeauth.csrf='))
  ?.split('=')[1];

await fetch('/api/auth/signout', {
  method: 'POST',
  headers: { 'x-csrf-token': csrf ?? '' },
});
```

Or simply use `useAuth().signOut()` from `@holeauth/react`, which handles the CSRF header for you.

Sign-out revokes the current refresh session and clears every auth cookie.

## Server components

```tsx
import { auth } from '@/lib/auth';
import { redirect } from 'next/navigation';

export default async function ProfilePage() {
  const session = await auth.getSession();
  if (!session) redirect('/login');

  const user = await db.user.findUnique({ where: { id: session.userId } });
  return <h1>Hello {user.name}</h1>;
}
```

`auth.getSession()` reads the access cookie from `next/headers` automatically when called without arguments inside a Server Component.
