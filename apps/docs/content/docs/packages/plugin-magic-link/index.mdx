# @holeauth/plugin-magic-link

Passwordless authentication: send a one-click magic link, a six-digit email OTP, or both. Works as the primary sign-in method or as a second factor on top of password.

## Installation

## Quick Example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { magicLink } from '@holeauth/plugin-magic-link';
import { createMagicLinkAdapter } from '@holeauth/magic-link-drizzle/pg';
import { db } from '@/db/client';
import { magicLinkSchema } from '@/db/schema';

const magicLinkAdapter = createMagicLinkAdapter({ db, tables: magicLinkSchema.tables });

export const auth = defineHoleauth({
  // ...core config
  plugins: [
    magicLink({
      adapter: magicLinkAdapter,
      baseUrl: 'https://example.com/api/auth',
      sendEmail: async ({ email, url, code, type }) => {
        if (type === 'magic-link') {
          await sendmail(email, `Sign in: ${url}`);
        } else {
          await sendmail(email, `Your code is ${code}`);
        }
      },
      mode: 'both',
      successRedirect: '/dashboard',
      errorRedirect: '/magic-link',
      expiredRedirect: '/magic-link/expired',
    }),
  ] as const,
});
```

## MagicLinkAdapter Interface

If you are not using [@holeauth/magic-link-drizzle](drizzle), implement this interface to provide custom storage.

```ts
interface MagicLinkAdapter {
  createToken(input: CreateMagicLinkInput): Promise<MagicLinkRecord>;
  findByTokenHash(tokenHash: string): Promise<MagicLinkRecord | null>;
  findActiveToken(identifier: string, type: MagicLinkTokenType): Promise<MagicLinkRecord | null>;
  atomicConsumeByHash(tokenHash: string): Promise<MagicLinkRecord | null>;
  atomicConsumeById(id: string): Promise<MagicLinkRecord | null>;
  deleteByIdentifier(identifier: string, type?: MagicLinkTokenType): Promise<void>;
  deleteByUserId(userId: string): Promise<void>;
  deleteExpired(): Promise<void>;
  findLatestByIdentifier(identifier: string, type: MagicLinkTokenType): Promise<MagicLinkRecord | null>;
}
```

## API Surface

The plugin merges its API onto `auth.magicLink.*`:

### request()

Send a magic link or OTP to the given email. Silently no-ops when the policy disallows the user (e.g. unknown email with `useFor: 'login'`).

```ts
auth.magicLink.request(input: {
  email: string;
  /** Override `options.mode`. Ignored when mode is fixed (not `'both'`). */
  type?: 'magic-link' | 'otp';
  ip?: string;
  userAgent?: string;
}): Promise<{ sent: boolean }>
```

**Throws**

- `MAGIC_LINK_INVALID_INPUT` — Email format invalid.
- `MAGIC_LINK_RATE_LIMITED` — Too many recent attempts (per email + IP).
- `MAGIC_LINK_RESEND_TOO_SOON` — `resendCooldownSeconds` not yet elapsed.

### consume()

Verify a magic-link token (read from URL query) and issue session tokens.

```ts
auth.magicLink.consume(input: {
  token: string;
  ip?: string;
  userAgent?: string;
}): Promise<{ user: AdapterUser; tokens: IssuedTokens }>
```

**Throws**

- `MAGIC_LINK_INVALID_INPUT` — Token format invalid.
- `MAGIC_LINK_NOT_FOUND` — Token does not exist.
- `MAGIC_LINK_ALREADY_USED` — Token already consumed.
- `MAGIC_LINK_EXPIRED` — Token past `expiresAt`.
- `MAGIC_LINK_EMAIL_NOT_VERIFIED` — Set only when `blockLoginBeforeEmailVerification === true`.

### verifyOtp()

Verify a numeric OTP code and issue session tokens.

```ts
auth.magicLink.verifyOtp(input: {
  email: string;
  code: string;
  ip?: string;
  userAgent?: string;
}): Promise<{ user: AdapterUser; tokens: IssuedTokens }>
```

**Throws**

- `MAGIC_LINK_INVALID_INPUT` — Code format invalid (non-digit or wrong length).
- `MAGIC_LINK_RATE_LIMITED` — Too many verify attempts.
- `MAGIC_LINK_INVALID_CODE` — Code did not match the active OTP.
- `MAGIC_LINK_EXPIRED` — Active OTP expired before verification.

## Modes

### Magic link only (default)

```ts
magicLink({ mode: 'magic-link', sendEmail, baseUrl, adapter })
```

The user enters their email, receives a URL, clicks it, and `GET /magic-link/consume?token=…` signs them in.

### OTP only

```ts
magicLink({ mode: 'otp', sendEmail, baseUrl, adapter })
```

The user enters their email, receives a six-digit code, types it into a verify form, and `POST /magic-link/verify-otp` signs them in. Useful for native apps and email clients that mangle long URLs.

### Both

```ts
magicLink({ mode: 'both', sendEmail, baseUrl, adapter })
```

The client picks per request by passing `{ type: 'magic-link' | 'otp' }` on `request()` (or the corresponding form field on `POST /magic-link/request`).

## Roles

### Primary

The plugin is a complete sign-in method. Successful `consume` / `verifyOtp` issues access + refresh tokens.

### Second factor

```ts
magicLink({ role: 'secondFactor', sendEmail, baseUrl, adapter })
```

Used in combination with another primary plugin (e.g. password). After the primary plugin signs in, the plugin emits a `signIn.challenge` instead of issuing tokens. The pending token is short-lived (`pendingTtlSeconds`). The client then collects an OTP or follows the email link to complete sign-in.

## Useful For

- `useFor: 'login'` — login-only; unknown emails return `{ sent: true }` but no email is sent (prevents account enumeration).
- `useFor: 'register'` — registration-only; existing emails return `{ sent: true }` but no email is sent. Registration is always enabled in this scope.
- `useFor: 'both'` — default; whether unknown emails are auto-registered is controlled by `allowRegistration`.

## Security

<Callout type="warn">
  Magic-link tokens and OTP codes are stored only as SHA-256 hashes (base64url). The raw value is in the email and in the URL the user clicks — never on the server. Even with full database access, an attacker cannot reuse a previously delivered token.
</Callout>

- **Constant-time comparison.** Hash lookup and OTP comparison both use constant-time string equality.
- **Single-use atomic consume.** `atomicConsumeByHash` marks the row used in one UPDATE statement; concurrent clicks of the same link can only succeed once.
- **Email enumeration protection.** All `useFor` policies return the same shape regardless of whether the email is known.
- **Per-email + per-IP rate limiting.** Defaults to 5 attempts per 5 minutes for both request and verify endpoints. Swap for a distributed limiter in production.
- **Cookie security.** Sets `Secure` cookies when `baseUrl` uses `https:`. Locked to `HttpOnly` and `SameSite=Lax`.

## Utility Exports

The plugin also exports:

```ts
import {
  createMemoryRateLimiter,
  generateMagicToken,
  generateOtp,
  hashToken,
  constantTimeEquals,
} from '@holeauth/plugin-magic-link';
```

- `createMemoryRateLimiter(options?)` — in-memory limiter for dev. Pass a Redis-backed implementation in production.
- `generateMagicToken()` — 32-byte cryptographically random token (base64url).
- `generateOtp(length)` — numeric OTP of the given length.
- `hashToken(token)` — SHA-256 hash (base64url).
- `constantTimeEquals(a, b)` — timing-safe string comparison.
