# @holeauth/adapter-drizzle

Drizzle ORM reference adapter for holeauth core. Generates all required holeauth tables (sessions, accounts, verification tokens, audit log) and returns a fully typed `HoleauthAdapterBundle` that plugs directly into `defineHoleauth`.

Supports **PostgreSQL**, **MySQL**, and **SQLite**.

## Quick example (PostgreSQL)

```ts title="db/schema.ts"
import { pgTable, text, timestamp } from 'drizzle-orm/pg-core';

export const users = pgTable('users', {
  id: text('id').primaryKey(),
  email: text('email').notNull().unique(),
  emailVerified: timestamp('email_verified', { withTimezone: true }),
  name: text('name'),
  image: text('image'),
  passwordHash: text('password_hash'),
});
```

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { createHoleauthTables, createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db';
import { users } from '@/db/schema';

const { tables } = createHoleauthTables({ usersTable: users });
const adapters = createHoleauthAdapters({ db, tables });

export const auth = defineHoleauth({
  adapter: adapters,
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
});
```

<Callout type="info">
  You own the `users` table. holeauth generates companion tables (sessions, accounts, verification tokens, audit log) prefixed with `holeauth_` and linked to your `users` table via foreign keys.
</Callout>

## `createHoleauthTables(options)`

Generate all holeauth companion table definitions. Call once and pass the result to `createHoleauthAdapters`.

### Options

<TypeTable
  type={{
    usersTable: {
      description: 'Your Drizzle users table. Must have a string-compatible `id` column.',
      type: 'PgUsersTable',
      required: true,
    },
    prefix: {
      description: 'Prefix prepended to every generated table name.',
      type: 'string',
      default: '"holeauth_"',
    },
  }}
/>

### Returns

```ts
{
  tables: {
    users: U;
    sessions: PgTable;            // holeauth_session
    accounts: PgTable;            // holeauth_account
    verificationTokens: PgTable;  // holeauth_verification_token
    auditLog: PgTable;            // holeauth_audit_log
  };
  relations: {
    sessionsRelations: Relations;
    accountsRelations: Relations;
    auditLogRelations: Relations;
  };
}
```

## `createHoleauthAdapters(options)`

Build the `HoleauthAdapterBundle` from the generated tables.

### Options

<TypeTable
  type={{
    db: {
      description: 'Drizzle database instance (postgres-js, node-postgres, or compatible).',
      type: 'PgDb',
      required: true,
    },
    tables: {
      description: 'Tables returned by createHoleauthTables.',
      type: 'HoleauthTables',
      required: true,
    },
    userEmailColumn: {
      description: 'Name of the email column on your users table.',
      type: 'string',
      default: '"email"',
    },
    generateId: {
      description: 'Custom ID generator for new records.',
      type: '() => string',
      default: 'crypto.randomUUID()',
    },
  }}
/>

### Returns `HoleauthAdapterBundle`

<TypeTable
  type={{
    user: { description: 'UserAdapter — CRUD operations for your users table.', type: 'UserAdapter', required: true },
    session: { description: 'SessionAdapter — session lifecycle (create, rotate, revoke, family queries).', type: 'SessionAdapter', required: true },
    account: { description: 'AccountAdapter — OAuth provider account links.', type: 'AccountAdapter', required: true },
    verificationToken: { description: 'VerificationTokenAdapter — email OTP / magic-link tokens.', type: 'VerificationTokenAdapter', required: true },
    auditLog: { description: 'AuditLogAdapter — append-only event log.', type: 'AuditLogAdapter', required: true },
    transaction: { description: 'TransactionAdapter — wraps mutations in a DB transaction.', type: 'TransactionAdapter', required: true },
  }}
/>

## Dialects

| Import path | Dialect |
|-------------|---------|
| `@holeauth/adapter-drizzle/pg` | PostgreSQL |
| `@holeauth/adapter-drizzle/mysql` | MySQL / PlanetScale |
| `@holeauth/adapter-drizzle/sqlite` | SQLite / Turso / Cloudflare D1 |

All three subpaths export identical `createHoleauthTables` and `createHoleauthAdapters` APIs with dialect-specific column types.

## Generated table schema

### `holeauth_session`

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | Session ID |
| `user_id` | text FK → users | Cascade delete |
| `family_id` | text | Refresh token family for reuse detection |
| `refresh_token_hash` | text UNIQUE | SHA-256 of the refresh token |
| `expires_at` | timestamp | When the refresh token expires |
| `created_at` | timestamp | Defaulted to now |
| `revoked_at` | timestamp | Null if active |
| `user_agent` | text | |
| `ip` | text | |

### `holeauth_account`

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | |
| `user_id` | text FK → users | Cascade delete |
| `provider` | text | e.g. `"google"`, `"github"` |
| `provider_account_id` | text | UNIQUE with provider |
| `email` | text | |
| `access_token` / `refresh_token` | text | |
| `expires_at` | timestamp | |
| `id_token` | text | |

### `holeauth_verification_token`

| Column | Type | Notes |
|--------|------|-------|
| `identifier` | text | Email address (composite PK) |
| `token` | text | Hashed token (composite PK) |
| `expires_at` | timestamp | |

### `holeauth_audit_log`

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | |
| `type` | text | [`HoleauthEventType`](/docs/api/events) |
| `user_id` | text FK → users | Set null on delete |
| `session_id` | text | |
| `at` | timestamp | Event time |
| `ip` / `user_agent` | text | |
| `data` | jsonb | Structured event payload |
