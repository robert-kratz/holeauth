# @holeauth/adapter-drizzle

Drizzle ORM adapter for holeauth core storage, supporting PostgreSQL, MySQL, and SQLite.

<Callout type="info">
  This package provides adapters for **core holeauth tables only** (users, sessions, accounts, verification tokens, audit log). For plugin-specific tables, see [@holeauth/2fa-drizzle](/packages/plugin-2fa/drizzle), [@holeauth/passkey-drizzle](/packages/plugin-passkey/drizzle), [@holeauth/rbac-drizzle](/packages/plugin-rbac/drizzle), and [@holeauth/idp-drizzle](/packages/plugin-idp/drizzle).
</Callout>

## Subpath Exports

This package provides dialect-specific entry points. **There is no main entry point** — you must import from a subpath:

- `@holeauth/adapter-drizzle/pg` — PostgreSQL
- `@holeauth/adapter-drizzle/mysql` — MySQL
- `@holeauth/adapter-drizzle/sqlite` — SQLite

All three expose the same API: `createHoleauthTables` and `createHoleauthAdapters`.

## Usage by Dialect

<Tabs items={['PostgreSQL', 'MySQL', 'SQLite']}>
  <Tab value="PostgreSQL">
    ```ts title="db/schema.ts"
    import { pgTable, text } from 'drizzle-orm/pg-core';
    import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';

    export const users = pgTable('users', {
      id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
      email: text('email').notNull().unique(),
      passwordHash: text('password_hash'),
      name: text('name'),
      image: text('image'),
      emailVerified: text('email_verified').$type<boolean>().default(false),
    });

    const holeauth = createHoleauthTables({ usersTable: users });

    // Re-export individual tables so drizzle-kit includes them in migrations.
    // drizzle-kit only detects top-level named exports — it will not recurse
    // into holeauth.tables automatically.
    export const { sessions, accounts, verificationTokens, auditLog } = holeauth.tables;
    ```

    ```ts title="lib/auth.ts"
    import { defineHoleauth } from '@holeauth/core';
    import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
    import { db } from '@/db';
    import { holeauth } from '@/db/schema';

    const adapters = createHoleauthAdapters({ db, tables: holeauth.tables });

    export const auth = defineHoleauth({
      adapter: adapters,
      secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
    });
    ```
  </Tab>

## API Reference

### createHoleauthTables()

Creates Drizzle table definitions for holeauth core storage.

```ts
function createHoleauthTables<U extends UsersTable>(
  options: CreateHoleauthTablesOptions<U>
): {
  tables: {
    users: U;
    sessions: ...;
    accounts: ...;
    verificationTokens: ...;
    auditLog: ...;
  };
  relations: {...};
}
```

<TypeTable
  type={{
    usersTable: {
      description: 'Your Drizzle users table. Must have an `id` column (any string-compatible type).',
      type: 'DrizzleTable',
      required: true,
    },
    prefix: {
      description: 'Prefix for all generated table names.',
      type: 'string',
      default: '"holeauth_"',
    },
  }}
/>

**Returns**

An object with:

- `tables.users` — Your provided users table (passed through unchanged).
- `tables.sessions` — Session storage with columns: `id` (PK), `userId` (FK cascade), `familyId`, `refreshTokenHash` (unique), `expiresAt`, `createdAt`, `revokedAt`, `userAgent`, `ip`.
- `tables.accounts` — OAuth / SSO account links: `id` (PK), `userId` (FK cascade), `provider`, `providerAccountId` (unique per provider), `email`, `accessToken`, `refreshToken`, `expiresAt`, `tokenType`, `scope`, `idToken`.
- `tables.verificationTokens` — Email verification / password reset tokens: `identifier`, `token` (composite PK), `expiresAt`.
- `tables.auditLog` — Audit trail: `id` (PK), `type`, `userId` (FK set null), `sessionId`, `at`, `ip`, `userAgent`, `data` (jsonb).
- `relations` — Drizzle relations between tables.

**Example**

<Callout type="warn">
  drizzle-kit detects tables by scanning top-level named exports from your schema file. `createHoleauthTables()` returns a plain object — spreading it into another object (`export const tables = { ...holeauth.tables }`) does **not** expose the individual table instances to drizzle-kit. You must destructure and re-export each table individually, or drizzle-kit will skip them and the tables will never be created in the database.
</Callout>

```ts title="db/schema.ts"
import { createHoleauthTables } from '@holeauth/adapter-drizzle/pg';
import { users } from './users';

export const holeauth = createHoleauthTables({
  usersTable: users,
  prefix: 'app_',
});

// Required: re-export each table as a named export for drizzle-kit.
export const { sessions, accounts, verificationTokens, auditLog } = holeauth.tables;
```

### createHoleauthAdapters()

Constructs a complete adapter object that satisfies holeauth's `HoleauthConfig.adapter` contract.

```ts
function createHoleauthAdapters(
  options: CreateHoleauthAdaptersOptions
): {
  user: UserAdapter;
  session: SessionAdapter;
  account: AccountAdapter;
  auditLog: AuditLogAdapter;
  verificationToken: VerificationTokenAdapter;
  transaction: TransactionAdapter;
}
```

<TypeTable
  type={{
    db: {
      description: 'Drizzle database instance (node-postgres / postgres-js / mysql2 / better-sqlite3).',
      type: 'DrizzleDB',
      required: true,
    },
    tables: {
      description: 'Tables object returned by `createHoleauthTables`.',
      type: 'Tables',
      required: true,
    },
    generateId: {
      description: 'Custom ID generator for sessions, accounts, audit log entries.',
      type: '() => string',
      default: 'crypto.randomUUID',
    },
  }}
/>

**Example**

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { createHoleauthAdapters } from '@holeauth/adapter-drizzle/pg';
import { db } from '@/db';
import { tables } from '@/db/schema';

const adapters = createHoleauthAdapters({ db, tables });

export const auth = defineHoleauth({
  adapter: adapters,
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
});
```

## Adapter Methods

The adapter object returned by `createHoleauthAdapters` implements the following interfaces:

### UserAdapter

```ts
interface UserAdapter {
  getUserById(id: string): Promise<AdapterUser | null>;
  getUserByEmail(email: string): Promise<AdapterUser | null>;
  createUser(data: {...}): Promise<AdapterUser>;
  updateUser(id: string, patch: Partial<AdapterUser>): Promise<AdapterUser>;
  deleteUser(id: string): Promise<void>;
}
```

### SessionAdapter

```ts
interface SessionAdapter {
  createSession(data: {...}): Promise<AdapterSession>;
  getSession(id: string): Promise<AdapterSession | null>;
  getByRefreshHash(hash: string): Promise<AdapterSession | null>;
  findByFamily(familyId: string): Promise<AdapterSession[]>;
  deleteSession(id: string): Promise<void>;
  rotateRefresh(id: string, newHash: string, expiresAt: Date): Promise<AdapterSession>;
  revokeFamily(familyId: string): Promise<void>;
  revokeUser(userId: string): Promise<void>;
}
```

### AccountAdapter

```ts
interface AccountAdapter {
  createAccount(data: {...}): Promise<AdapterAccount>;
  getAccountByProvider(provider: string, providerAccountId: string): Promise<AdapterAccount | null>;
  listAccountsByUserId(userId: string): Promise<AdapterAccount[]>;
  updateAccount(id: string, patch: Partial<AdapterAccount>): Promise<AdapterAccount>;
  deleteAccount(id: string): Promise<void>;
}
```

### VerificationTokenAdapter

```ts
interface VerificationTokenAdapter {
  createToken(data: {...}): Promise<void>;
  getToken(identifier: string, token: string): Promise<AdapterVerificationToken | null>;
  deleteToken(identifier: string, token: string): Promise<void>;
  deleteExpiredTokens(): Promise<number>;
}
```

### AuditLogAdapter

```ts
interface AuditLogAdapter {
  createEntry(data: {...}): Promise<AdapterAuditEvent>;
  listEntries(opts?: {userId?: string; type?: string; limit?: number}): Promise<AdapterAuditEvent[]>;
}
```

### TransactionAdapter

```ts
interface TransactionAdapter {
  transaction<T>(fn: (tx: unknown) => Promise<T>): Promise<T>;
}
```

Wraps a callback in a database transaction. The `tx` object passed to `fn` is a Drizzle transaction instance.

## Schema Details

### Sessions Table (PostgreSQL example)

| Column | Type | Constraints |
| --- | --- | --- |
| `id` | `text` | Primary key |
| `userId` | `text` | Not null, foreign key → `users.id` (on delete cascade), indexed |
| `familyId` | `text` | Not null, indexed (for refresh token rotation) |
| `refreshTokenHash` | `text` | Not null, unique (SHA-256 hash) |
| `expiresAt` | `timestamp` | Not null |
| `createdAt` | `timestamp` | Not null, default now |
| `revokedAt` | `timestamp` | Nullable (set when session is revoked) |
| `userAgent` | `text` | Nullable |
| `ip` | `text` | Nullable |

### Accounts Table

| Column | Type | Constraints |
| --- | --- | --- |
| `id` | `text` | Primary key |
| `userId` | `text` | Not null, foreign key → `users.id` (on delete cascade), indexed |
| `provider` | `text` | Not null |
| `providerAccountId` | `text` | Not null |
| `email` | `text` | Nullable |
| `accessToken` | `text` | Nullable |
| `refreshToken` | `text` | Nullable |
| `expiresAt` | `timestamp` | Nullable |
| `tokenType` | `text` | Nullable |
| `scope` | `text` | Nullable |
| `idToken` | `text` | Nullable |

Unique constraint: `(provider, providerAccountId)`.

### Verification Tokens Table

| Column | Type | Constraints |
| --- | --- | --- |
| `identifier` | `text` | Part of composite PK (usually email) |
| `token` | `text` | Part of composite PK (random token) |
| `expiresAt` | `timestamp` | Not null |

### Audit Log Table

| Column | Type | Constraints |
| --- | --- | --- |
| `id` | `text` | Primary key |
| `type` | `text` | Not null, indexed |
| `userId` | `text` | Nullable, foreign key → `users.id` (on delete set null), indexed |
| `sessionId` | `text` | Nullable |
| `at` | `timestamp` | Not null, default now |
| `ip` | `text` | Nullable |
| `userAgent` | `text` | Nullable |
| `data` | `jsonb` | Nullable (additional structured data) |

## Headless Alternative

If you are not using Drizzle, implement the adapter interfaces manually. See the [Adapters concept page](/concepts/adapters) for the full contract.

**Example (custom adapter)**

```ts title="lib/my-adapter.ts"
import { defineHoleauth } from '@holeauth/core';
import type { UserAdapter, SessionAdapter } from '@holeauth/core/adapters';

const userAdapter: UserAdapter = {
  async getUserById(id) {
    return myDb.query('SELECT * FROM users WHERE id = $1', [id]);
  },
  // ... implement all methods
};

const sessionAdapter: SessionAdapter = {
  // ... implement all methods
};

export const auth = defineHoleauth({
  adapter: { user: userAdapter, session: sessionAdapter, /* ... */ },
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
});
```

## Migration

After defining your schema, generate and run a migration:

```bash
npx drizzle-kit generate
npx drizzle-kit migrate
```

<Callout type="warn">
  Do not rename holeauth-managed columns. The adapter expects exact column names as defined by `createHoleauthTables`. You can add custom columns to your users table.
</Callout>
