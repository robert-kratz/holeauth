# @holeauth/rbac-yaml

Load RBAC group definitions from a YAML file with optional hot-reload. Node.js only.

## Installation

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
```bash

```
</Tab>
  <Tab value="npm">

```bash
npm install @holeauth/rbac-yaml
```

</Tab>
  <Tab value="yarn">

```bash
yarn add @holeauth/rbac-yaml
```
</Tab>
  <Tab value="bun">
```bash
bun add @holeauth/rbac-yaml
```

<Callout type="warn">
  This package depends on Node.js `fs` and `path` APIs. It is **NOT** compatible with edge runtimes (Cloudflare Workers, Vercel Edge, Deno Deploy). For edge environments, use `loadRbacYamlFromString` and supply the YAML content from an environment variable or KV store.
</Callout>

## Quick Example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { rbac } from '@holeauth/plugin-rbac';
import { loadRbacYaml } from '@holeauth/rbac-yaml';
import { createRbacAdapter } from '@holeauth/rbac-drizzle/pg';
import { db } from '@/db';
import { tables } from '@/db/schema';

const yaml = loadRbacYaml('./holeauth.rbac.yml', {
  watch: process.env.NODE_ENV !== 'production',
});

const plugin = rbac({
  groups: yaml.snapshot.groups,
  adapter: createRbacAdapter({ db, tables }),
});

yaml.onReload((snapshot) => plugin.reload(snapshot.groups));

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  plugins: [plugin],
});
```

```yaml title="holeauth.rbac.yml"
groups:
  user:
    default: true
    displayName: User
    priority: 100
    permissions:
      - posts.read
      - comments.create
      - comments.read

  editor:
    displayName: Editor
    priority: 50
    inherits:
      - user
    permissions:
      - posts.create
      - posts.update
      - posts.*.self

  admin:
    displayName: Administrator
    priority: 0
    permissions:
      - '*'
```

## API Reference

### loadRbacYaml()

Loads and validates a YAML file from the filesystem. Returns a snapshot object with optional file-watch.

```ts

  path: string,
  options?: LoadYamlOptions
```

**Example**

```ts title="lib/auth.ts"
import { loadRbacYaml } from '@holeauth/rbac-yaml';

const yaml = loadRbacYaml('./holeauth.rbac.yml', {
  watch: process.env.NODE_ENV === 'development',
  multipleDefaults: 'throw',
});

console.log(yaml.snapshot.groups);

yaml.onReload((snapshot) => {
  console.log('RBAC config reloaded:', snapshot.groups.length, 'groups');
});
```

### validateRbacDefinition()

Validates a JavaScript object (already parsed from YAML or JSON) against the schema.

```ts
function validateRbacDefinition(
  obj: unknown,
  options?: LoadOptions
): RbacConfigSnapshot
```

**Throws**

- Zod validation errors if the structure is invalid.
- `Error` if no default group or multiple defaults (depending on `multipleDefaults` option).
- `Error` on inheritance cycles.

## YAML Schema

The YAML file must have a top-level `groups` object. Each key is a group ID.

### Group Properties

<TypeTable
  type={{
    default: {
      description: 'Whether this is the default group assigned to new users. Exactly one group must have `default: true`.',
      type: 'boolean',
    },
    displayName: {
      description: 'Human-readable group name.',
      type: 'string',
    },
    description: {
      description: 'Group description.',
      type: 'string',
    },
    priority: {
      description: 'Sort order for permission precedence (lower = higher priority).',
      type: 'number',
    },
    inherits: {
      description: 'Array of group IDs to inherit permissions from.',
      type: 'string[]',
    },
    permissions: {
      description: 'Array of permission nodes (dot-separated, wildcard `*`, negation prefix `!`).',
      type: 'string[]',
    },
  }}
/>

### Permission Node Syntax

- `posts.create` — Exact match
- `posts.*` — Wildcard (matches `posts.create`, `posts.delete`, etc.)
- `*` — Global wildcard (matches everything)
- `!posts.delete` — Negation (revokes a permission)

### Inheritance

Groups can inherit permissions from other groups using the `inherits` field. Inheritance is resolved at load time, and the result is stored in the `effective` array.

**Example**

```yaml
groups:
  user:
    default: true
    permissions:
      - posts.read
      - comments.create

  moderator:
    inherits:
      - user
    permissions:
      - comments.delete

  admin:
    inherits:
      - moderator
    permissions:
      - '*'
```

The `admin` group's effective permissions are: `['posts.read', 'comments.create', 'comments.delete', '*']`.

**Circular inheritance is detected and raises an error.**

## Complete Example

```yaml title="holeauth.rbac.yml"
groups:
  # Default group for all users
  user:
    default: true
    displayName: User
    description: Standard user with read access
    priority: 100
    permissions:
      - posts.read
      - comments.create
      - comments.read
      - profile.update.self

  # Content creators
  creator:
    displayName: Content Creator
    description: Can create and manage their own posts
    priority: 75
    inherits:
      - user
    permissions:
      - posts.create
      - posts.update.self
      - posts.delete.self

  # Moderators
  moderator:
    displayName: Moderator
    description: Can moderate comments and posts
    priority: 50
    inherits:
      - creator
    permissions:
      - comments.delete
      - posts.flag
      - reports.read

  # Editors
  editor:
    displayName: Editor
    description: Can edit all content
    priority: 25
    inherits:
      - moderator
    permissions:
      - posts.*
      - '!posts.delete'  # Can edit but not delete

  # Administrators
  admin:
    displayName: Administrator
    description: Full access to everything
    priority: 0
    permissions:
      - '*'
```

## Hot Reload

When `watch: true` is enabled, the loader monitors the YAML file for changes and automatically reloads the configuration.

```ts title="lib/auth.ts"
import { loadRbacYaml } from '@holeauth/rbac-yaml';
import { rbac } from '@holeauth/plugin-rbac';

const yaml = loadRbacYaml('./holeauth.rbac.yml', {
  watch: process.env.NODE_ENV === 'development',
});

const plugin = rbac({
  groups: yaml.snapshot.groups,
  adapter: rbacAdapter,
});

yaml.onReload((snapshot) => {
  console.log('[RBAC] Config reloaded:', snapshot.groups.length, 'groups');
  plugin.reload(snapshot.groups);
});

// Clean up when your app shuts down
process.on('SIGTERM', () => yaml.close());
```

## Validation Errors

The loader uses Zod for validation. Common errors:

- **No default group**: At least one group must have `default: true`.
- **Multiple defaults**: Only one group can be the default (configurable via `multipleDefaults` option).
- **Inheritance cycle**: Groups form a circular inheritance chain.
- **Unknown parent**: A group inherits from a non-existent group ID.
- **Invalid permission syntax**: Permission nodes must match `/^!?[A-Za-z0-9_\-]+(\.([A-Za-z0-9_\-]+|\*))*\*?$|^\*$/`.
