# @holeauth/plugin-idp

Run an OpenID Connect / OAuth 2.0 Identity Provider with Authorization Code + PKCE, refresh token rotation, and multi-tenant app management.

<Callout type="info">
  This plugin turns your holeauth instance into an **OIDC Provider / OAuth 2.0 Authorization Server**. To consume an external OIDC provider (e.g., Google, GitHub), use [@holeauth/plugin-idp-consumer](/packages/plugin-idp-consumer) instead.
</Callout>

## Quick Example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { idp } from '@holeauth/plugin-idp';
import { createIdpAdapter } from '@holeauth/idp-drizzle/pg';
import { db } from '@/db';
import { tables } from '@/db/schema';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  plugins: [
    idp({
      adapter: createIdpAdapter({ db, tables }),
      issuer: 'https://auth.example.com',
      scopesSupported: ['openid', 'profile', 'email', 'offline_access'],
      accessTokenTtl: 900,
      refreshTokenTtl: 2592000,
    }),
  ],
});
```

**Example**

```ts title="lib/auth.ts"
import { idp } from '@holeauth/plugin-idp';
import { createIdpAdapter } from '@holeauth/idp-drizzle/pg';
import { createRedisRateLimiter } from '@/lib/rate-limiter';
import { db } from '@/db';
import { tables } from '@/db/schema';

const plugin = idp({
  adapter: createIdpAdapter({ db, tables }),
  issuer: 'https://auth.acme.com',
  scopesSupported: ['openid', 'profile', 'email', 'offline_access', 'api.read', 'api.write'],
  accessTokenTtl: 1800,
  idTokenTtl: 3600,
  refreshTokenTtl: 7776000,
  authorizationCodeTtl: 300,
  signingAlg: 'RS256',
  tokenRateLimiter: createRedisRateLimiter(),
});
```

## IdpAdapter Interface

If you are not using [@holeauth/idp-drizzle](drizzle), implement this interface to provide custom storage.

```ts
interface IdpAdapter {
  apps: {
    create(data: {...}): Promise<IdpApp>;
    get(id: string): Promise<IdpApp | null>;
    list(): Promise<IdpApp[]>;
    listByTeam(teamId: string): Promise<IdpApp[]>;
    update(id: string, patch: {...}): Promise<void>;
    delete(id: string): Promise<void>;
  };
  teams: {
    create(data: {...}): Promise<IdpTeam>;
    get(id: string): Promise<IdpTeam | null>;
    listForUser(userId: string): Promise<Array<{...}>>;
    getMembership(teamId: string, userId: string): Promise<IdpTeamMember | null>;
    listMembers(teamId: string): Promise<IdpTeamMember[]>;
    addMember(data: {...}): Promise<void>;
    removeMember(teamId: string, userId: string): Promise<void>;
  };
  authorizationCodes: {
    create(data: {...}): Promise<void>;
    getByCodeHash(codeHash: string): Promise<IdpAuthorizationCode | null>;
    consume(codeHash: string): Promise<void>;
    purgeExpired(): Promise<number>;
  };
  refreshTokens: {
    create(data: {...}): Promise<IdpRefreshToken>;
    getByTokenHash(hash: string): Promise<IdpRefreshToken | null>;
    revoke(id: string): Promise<void>;
    revokeFamily(familyId: string): Promise<void>;
    listForApp(appId: string): Promise<IdpRefreshToken[]>;
    revokeAllForApp(appId: string): Promise<void>;
    purgeExpired(): Promise<number>;
  };
  consents: {
    upsert(data: {...}): Promise<void>;
    get(userId: string, appId: string): Promise<IdpConsent | null>;
    list(userId: string): Promise<IdpConsent[]>;
    revoke(userId: string, appId: string): Promise<void>;
  };
  keys: {
    create(data: {...}): Promise<IdpSigningKey>;
    getCurrent(alg: SigningAlg): Promise<IdpSigningKey | null>;
    list(alg: SigningAlg): Promise<IdpSigningKey[]>;
    delete(kid: string): Promise<void>;
  };
}
```

## API Surface

The plugin exposes the following methods on your auth instance at `auth.idp.*`:

### apps.create()

Creates a new OAuth application in a team.

```ts
auth.idp.apps.create(
  callerUserId: string,
  input: {
    name: string;
    description?: string | null;
    type: 'confidential' | 'public';
    redirectUris: string[];
    allowedScopes?: string[];
    requirePkce?: boolean;
    teamId?: string;
  }
): Promise<{ app: IdpApp; clientSecret?: string }>
```

<Callout type="warn">
  The `clientSecret` is returned **only once** for confidential apps. It is hashed before storage and cannot be retrieved later.
</Callout>

### apps.listForUser()

Lists all apps the user has access to via team membership.

```ts
auth.idp.apps.listForUser(userId: string): Promise<IdpApp[]>
```

### apps.get()

Fetches an app by ID. Caller must be a team member (or have admin permission).

```ts
auth.idp.apps.get(
  callerUserId: string,
  appId: string,
  opts?: { admin?: boolean }
): Promise<IdpApp>
```

### apps.update()

Updates an app's configuration.

```ts
auth.idp.apps.update(
  callerUserId: string,
  appId: string,
  patch: {
    name?: string;
    description?: string | null;
    logoUrl?: string | null;
    redirectUris?: string[];
    allowedScopes?: string[];
    requirePkce?: boolean;
    disabled?: boolean;
  },
  opts?: { admin?: boolean }
): Promise<IdpApp>
```

### apps.regenerateSecret()

Regenerates the `client_secret` for a confidential app. Returns the plain-text secret once.

```ts
auth.idp.apps.regenerateSecret(
  callerUserId: string,
  appId: string
): Promise<{ clientSecret: string }>
```

### apps.delete()

Deletes an OAuth app.

```ts
auth.idp.apps.delete(
  callerUserId: string,
  appId: string,
  opts?: { admin?: boolean }
): Promise<void>
```

### teams.create()

Creates a team with the caller as owner.

```ts
auth.idp.teams.create(ownerUserId: string, name: string): Promise<IdpTeam>
```

### teams.listForUser()

Lists all teams the user belongs to, with their role.

```ts
auth.idp.teams.listForUser(userId: string): Promise<Array<IdpTeam & { role: TeamRole }>>
```

### teams.addMember()

Adds a user to a team with a specific role (`owner` or `developer`).

```ts
auth.idp.teams.addMember(
  callerUserId: string,
  teamId: string,
  userId: string,
  role: 'owner' | 'developer'
): Promise<void>
```

### teams.removeMember()

Removes a user from a team.

```ts
auth.idp.teams.removeMember(
  callerUserId: string,
  teamId: string,
  userId: string
): Promise<void>
```

### tokens.listForApp()

Lists all active refresh tokens for an app.

```ts
auth.idp.tokens.listForApp(
  callerUserId: string,
  appId: string
): Promise<IdpRefreshToken[]>
```

### tokens.revokeAllForApp()

Revokes all refresh tokens for an app (e.g., after a security breach).

```ts
auth.idp.tokens.revokeAllForApp(
  callerUserId: string,
  appId: string
): Promise<void>
```

### keys.rotate()

Rotates the signing key for access and ID tokens. Returns the new key.

```ts
auth.idp.keys.rotate(): Promise<IdpSigningKey>
```

### keys.bootstrap()

Ensures a signing key exists. Called automatically during plugin initialization.

```ts
auth.idp.keys.bootstrap(): Promise<IdpSigningKey>
```

## OAuth 2.0 Flows

### Authorization Code + PKCE

1. Relying Party redirects user to `/oauth2/authorize?...`.
2. User authenticates and consents (or consent is remembered).
3. Server issues an authorization code (10-minute TTL).
4. RP exchanges the code for tokens at `/oauth2/token`.
5. Access token (JWT, 15-minute TTL) and refresh token (30-day TTL) are returned.

### Refresh Token Rotation

Every refresh token has a `familyId`. When a token is refreshed:

- The old token is revoked.
- A new token is issued with the same `familyId`.
- If an old token is reused, the entire family is revoked (reuse detection).

This matches the holeauth core session refresh strategy.

## Team-Based App Ownership

OAuth apps are owned by **teams**, not individual users. A team has:

- **Owners**: Full control (create/update/delete apps, manage members).
- **Developers**: Can view and test apps, but cannot modify settings.

Users can belong to multiple teams and create apps in any team they own.

## OIDC Scopes

The plugin enforces the standard OIDC scopes during the authorization code, refresh, and userinfo flows. Requested scopes are intersected with the registered `allowedScopes` on each `IdpApp`; the intersection is what gets written into the access token, refresh token, and persisted consent.

### BUILTIN_SCOPES

```ts
const BUILTIN_SCOPES = ['openid', 'profile', 'email', 'offline_access'] as const;
```

| Scope | Effect |
| --- | --- |
| `openid` | Required for any OIDC flow. Triggers issuance of an `id_token`. |
| `profile` | Adds `name`, `preferred_username`, and `picture` to the ID token and `userinfo` response. |
| `email` | Adds `email` and `email_verified`. |
| `offline_access` | Authorises issuance of a refresh token. |

Custom scopes are passed through unchanged — they appear in the access token's `scope` claim and in the consent record, but do not contribute extra ID-token claims unless your resource server reads them.

### Scope Helpers

The plugin exports four pure helpers for inspecting and combining scopes outside the standard flows — useful in custom routes, audit dashboards, or admin tooling.

```ts
import {
  parseScope,
  formatScope,
  intersectScopes,
  claimsForUser,
} from '@holeauth/plugin-idp';
```

| Function | Signature | Use |
| --- | --- | --- |
| `parseScope` | `(s: string \| null) => string[]` | Split a space-separated scope string into an array. |
| `formatScope` | `(scopes: string[]) => string` | Join an array into a space-separated string, de-duplicating. |
| `intersectScopes` | `(requested: string[], allowed: string[]) => string[]` | Intersect requested scopes with an app's `allowedScopes`, preserving request order. |
| `claimsForUser` | `(user: AdapterUser, scopes: string[]) => Record<string, unknown>` | Compute the OIDC profile/email claims that the plugin emits for a given user and scope set. |

```ts title="app/admin/apps/[id]/route.ts"
import { intersectScopes, parseScope, formatScope } from '@holeauth/plugin-idp';

const granted = intersectScopes(parseScope(req.query.scope), app.allowedScopes);
return Response.json({ granted: formatScope(granted) });
```

<Callout type="info">
  `claimsForUser` is the same helper the plugin uses internally for the ID token and the `userinfo` response. Calling it directly is the supported way to mirror ID-token contents inside a custom route — for example, when exposing a `/me` endpoint that does not run through `userinfo`.
</Callout>
