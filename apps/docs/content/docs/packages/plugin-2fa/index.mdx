# @holeauth/plugin-2fa

Add Time-Based One-Time Password (TOTP) two-factor authentication with recovery codes to your holeauth application.

## Quick Example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { twofa } from '@holeauth/plugin-2fa';
import { createTwoFactorAdapter } from '@holeauth/2fa-drizzle/pg';
import { db } from '@/db';
import { tables } from '@/db/schema';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db),
  secrets: { jwtSecret: process.env.HOLEAUTH_SECRET! },
  plugins: [
    twofa({
      adapter: createTwoFactorAdapter({ db, tables }),
      issuer: 'My App',
      recoveryCodeCount: 12,
    }),
  ],
});
```

## API Surface

The plugin exposes the following methods on your auth instance at `auth.twofa.*`:

### setup()

Generates a new TOTP secret, saves it (disabled), and returns the `otpauth://` URI plus a data-URL-encoded QR code.

```ts
auth.twofa.setup(userId: string): Promise<{
  secret: string;
  otpauthUrl: string;
  qrUrl: string;
}>
```

**Example**

```ts title="app/api/2fa/setup/route.ts"
import { auth } from '@/lib/auth';

export async function POST(req: Request) {
  const session = await auth.validateCurrentRequest(req);
  if (!session) return new Response('Unauthorized', { status: 401 });

  const result = await auth.twofa.setup(session.userId);
  return Response.json(result);
}
```

### activate()

Verifies a live TOTP code and enables 2FA. Returns an array of recovery codes (plain-text, **shown once**).

```ts
auth.twofa.activate(userId: string, code: string): Promise<{
  recoveryCodes: string[];
}>
```

<Callout type="warn">
  Recovery codes are single-use and stored hashed. Display them to the user immediately and prompt them to save them securely. They are **not** retrievable afterwards — use `regenerateRecoveryCodes()` to issue a fresh set.
</Callout>

**Example**

```ts title="app/api/2fa/activate/route.ts"
import { auth } from '@/lib/auth';

export async function POST(req: Request) {
  const session = await auth.validateCurrentRequest(req);
  if (!session) return new Response('Unauthorized', { status: 401 });

  const { code } = await req.json();
  const { recoveryCodes } = await auth.twofa.activate(session.userId, code);

  return Response.json({ recoveryCodes });
}
```

### verify()

Consumes a pending challenge token (issued during sign-in when 2FA is enabled) and verifies the provided code (TOTP or recovery). Returns session tokens on success.

```ts
auth.twofa.verify(input: {
  pendingToken: string;
  code: string;
  ip?: string;
  userAgent?: string;
}): Promise<{ user: AdapterUser; tokens: IssuedTokens }>
```

**Throws**

- `TWOFA_INVALID_INPUT` — Code format is invalid.
- `TWOFA_RATE_LIMITED` — Too many attempts.
- `TWOFA_INVALID_CODE` — Code does not match or has already been used.

### disable()

Disables 2FA for the user. Requires a valid current code for verification.

```ts
auth.twofa.disable(userId: string, code: string): Promise<void>
```

### regenerateRecoveryCodes()

Replaces the user's existing recovery codes with a fresh set. Requires a valid live TOTP code to confirm intent and prevent silent account takeover if a session is stolen.

```ts
auth.twofa.regenerateRecoveryCodes(userId: string, code: string): Promise<{
  recoveryCodes: string[];
}>
```

**Throws**

- `TWOFA_NOT_ENROLLED` — User has no 2FA record.
- `INVALID_CREDENTIALS` — 2FA is not yet activated, or the TOTP code is wrong.
- `TWOFA_RATE_LIMITED` — Too many attempts.

**Example**

```ts title="app/settings/2fa/regenerate/route.ts"
import { auth } from '@/lib/auth';

export async function POST(req: Request) {
  const session = await auth.validateCurrentRequest(req);
  if (!session) return new Response('Unauthorized', { status: 401 });

  const { code } = await req.json();
  const { recoveryCodes } = await auth.twofa.regenerateRecoveryCodes(session.userId, code);

  return Response.json({ recoveryCodes });
}
```

<Callout type="warn">
  Old codes are invalidated immediately when this method succeeds. Show the new codes to the user exactly once and prompt them to save the file.
</Callout>

### isEnabled()

Checks if 2FA is currently active for the user.

```ts
auth.twofa.isEnabled(userId: string): Promise<boolean>
```

## Challenge Flow

When a user with 2FA enabled attempts to sign in:

1. `signIn` emits a challenge instead of issuing session tokens.
2. The challenge result includes a `pendingToken` cookie.
3. The client must collect a TOTP or recovery code and POST to `/2fa/verify`.
4. On success, session tokens replace the pending token.

## Utility Exports

The plugin also exports these helper functions:

### generateRecoveryCodes()

```ts
function generateRecoveryCodes(count: number): string[]
```

Generates an array of cryptographically random recovery codes (14 characters, formatted as `XXXX-XXXX-XXXX`).

### verifyTotp()

```ts
function verifyTotp(secret: string, code: string): boolean
```

Verifies a TOTP code against a secret (with ±1 window).

### buildOtpauthUrl()

```ts
function buildOtpauthUrl(options: {
  secret: string;
  issuer: string;
  label: string;
}): string
```

Constructs an `otpauth://totp/...` URI for QR code generation.

### renderQrDataUrl()

```ts
function renderQrDataUrl(payload: string): Promise<string>
```

Renders any payload (typically an `otpauth://` URI) as a PNG data URL for inline display.

### renderQrBuffer()

```ts
function renderQrBuffer(payload: string): Promise<Buffer>
```

Renders a payload as a raw PNG buffer (useful for serving QR codes from an API route).

### createMemoryRateLimiter()

Creates an in-memory rate limiter for development. Not recommended for production (no shared state across instances).

#### Distributed rate limiting

The plugin accepts any `TwoFactorRateLimiter` implementation. For multi-instance deployments back the limiter with a shared store such as Redis.

```ts title="lib/rate-limit.ts"
import type { TwoFactorRateLimiter } from '@holeauth/plugin-2fa';
import { redis } from './redis';

export function createRedisRateLimiter(
  maxAttempts = 5,
  windowSeconds = 300,
): TwoFactorRateLimiter {
  return {
    async hit(key) {
      const k = `2fa:${key}`;
      const count = await redis.incr(k);
      if (count === 1) await redis.expire(k, windowSeconds);
      return { allowed: count <= maxAttempts, remaining: Math.max(0, maxAttempts - count) };
    },
    async reset(key) {
      await redis.del(`2fa:${key}`);
    },
  };
}
```

Pass the limiter to the plugin via `twofa({ rateLimiter: createRedisRateLimiter() })`.

## Recovery Codes

Recovery codes are issued once during `activate()` and persisted on the user account. The plugin exposes three pure helpers — independent of the auth instance — that take a code array and prepare it for delivery to the user.

### formatRecoveryCodesAsText()

Format codes as a human-friendly text document with a heading, metadata block, and one code per line. Pure and runtime-agnostic.

```ts
function formatRecoveryCodesAsText(
  codes: readonly string[],
  options?: RecoveryCodesTxtOptions,
): string
```

### recoveryCodesToBlob()

Wrap `formatRecoveryCodesAsText()` output in a `Blob` (`text/plain;charset=utf-8`). Works in browsers and Node ≥ 18.

```ts
function recoveryCodesToBlob(
  codes: readonly string[],
  options?: RecoveryCodesTxtOptions,
): Blob
```

### downloadRecoveryCodesAsTxt()

Browser-only helper that triggers a file download. Call it from a click handler immediately after the server returns freshly-generated recovery codes — the user should never see them twice.

```ts
function downloadRecoveryCodesAsTxt(
  codes: readonly string[],
  options?: DownloadRecoveryCodesOptions,
): void
```

<TypeTable
  type={{
    fileName: {
      description: 'Filename presented to the user.',
      type: 'string',
      default: '"recovery-codes.txt"',
    },
  }}
/>

Inherits all `RecoveryCodesTxtOptions` fields. Throws when called outside a browser context.

```tsx title="app/account/2fa/RecoveryCodes.tsx"
'use client';
import { downloadRecoveryCodesAsTxt } from '@holeauth/plugin-2fa';

export function RecoveryCodes({ codes, email }: { codes: string[]; email: string }) {
  return (
    <>
      <ul>{codes.map((c) => <li key={c}><code>{c}</code></li>)}</ul>
      <button
        onClick={() =>
          downloadRecoveryCodesAsTxt(codes, {
            accountLabel: email,
            issuer: 'My App',
            fileName: 'my-app-recovery-codes.txt',
          })
        }
      >
        Download as .txt
      </button>
    </>
  );
}
```

<Callout type="warn">
  Recovery codes are shown exactly once. The server stores only their hashes — if the user loses the file they must rotate 2FA via `disable` + `setup`, or use `regenerateRecoveryCodes()`.
</Callout>

## React Hooks (`@holeauth/react`)

### use2faSetup()

Manages the full enrolment flow. The `activate()` function now surfaces `recoveryCodes` directly so you can display or download them in one step.

```tsx title="app/settings/2fa/EnrollPage.tsx"
'use client';
import { use2faSetup } from '@holeauth/react';
import { downloadRecoveryCodesAsTxt } from '@holeauth/plugin-2fa';

export function EnrollPage() {
  const { start, activate, data, loading, error } = use2faSetup();

  async function handleActivate(code: string) {
    const result = await activate(code);
    if (result.ok) {
      // recoveryCodes are included in the response
      downloadRecoveryCodesAsTxt(result.recoveryCodes, { issuer: 'My App' });
    }
  }

  return (/* … */);
}
```

**Returns**

| Field | Type | Description |
| --- | --- | --- |
| `start` | `() => Promise<TwoFaSetupResult \| null>` | Calls `POST /2fa/setup`, stores QR + secret. |
| `activate` | `(code: string) => Promise<{ ok: true; recoveryCodes: string[] } \| { ok: false }>` | Calls `POST /2fa/activate`, enables 2FA, returns recovery codes. |
| `data` | `TwoFaSetupResult \| null` | The latest setup payload (`qrUrl`, `secret`). |
| `loading` | `boolean` | True while a request is in flight. |
| `error` | `HoleauthErrorShape \| null` | Last error, if any. |

### use2faVerify()

Completes sign-in after a 2FA challenge. Pass the `pendingToken` from the sign-in challenge result.

```tsx title="app/2fa/verify/VerifyPage.tsx"
'use client';
import { use2faVerify } from '@holeauth/react';

export function VerifyPage({ pendingToken }: { pendingToken: string }) {
  const { verify, loading, error } = use2faVerify();
  return (/* … */);
}
```

### use2faRegenerateCodes()

Replaces the user's existing recovery codes with a fresh set. Requires the user to confirm with a live TOTP code.

```tsx title="app/settings/2fa/RegeneratePage.tsx"
'use client';
import { use2faRegenerateCodes } from '@holeauth/react';
import { downloadRecoveryCodesAsTxt } from '@holeauth/plugin-2fa';

export function RegeneratePage() {
  const { regenerate, loading, error, recoveryCodes } = use2faRegenerateCodes();

  async function handleSubmit(code: string) {
    const result = await regenerate(code);
    if (result.ok) {
      downloadRecoveryCodesAsTxt(result.recoveryCodes, { issuer: 'My App' });
    }
  }

  return (/* … */);
}
```

**Returns**

| Field | Type | Description |
| --- | --- | --- |
| `regenerate` | `(code: string) => Promise<{ ok: true; recoveryCodes: string[] } \| { ok: false }>` | Calls `POST /2fa/regenerate-recovery-codes`. |
| `recoveryCodes` | `string[] \| null` | The newly generated codes after a successful call. |
| `loading` | `boolean` | True while the request is in flight. |
| `error` | `HoleauthErrorShape \| null` | Last error, if any. |
