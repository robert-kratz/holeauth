# @holeauth/core

Edge-native authentication primitives. This package provides JWT signing and verification, session management with refresh-token rotation, password hashing with Argon2, OTP generation and verification, SSO consumer providers, adapter interfaces, plugin system, event bus, and error classes.

## Quick example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { drizzleAdapter } from '@holeauth/adapter-drizzle/pg';
import { db } from './db';

export const auth = defineHoleauth({
  adapter: drizzleAdapter(db),
  secrets: {
    jwtSecret: process.env.HOLEAUTH_SECRET!,
  },
  session: {
    accessTokenTTL: '15m',
    refreshTokenTTL: '7d',
  },
});
```

    jwtSecret: process.env.HOLEAUTH_SECRET!,

```

The returned `HoleauthInstance` exposes:

- `getSession(accessToken)` — Validate and decode an access token
- `flows.*` — High-level auth flows (register, signIn, signOut, refresh)
- `jwt.*` — JWT signing and verification
- `session.*` — Session CRUD (issue, rotate, revoke, validate)
- `password.*` — Password hashing and verification
- `otp.*` — OTP generation and verification
- `sso.*` — SSO consumer methods (authorize, callback)
- `adapters.*` — Adapter interfaces for custom implementations
- `cookies.*` — Cookie configuration helpers
- `events.*` — Event emitter and listener registration
- `errors.*` — Error classes (InvalidCredentialsError, InvalidTokenError, etc.)
- Plugin APIs when plugins are registered

const token = await sign(

  { expiresIn: '1h', issuer: 'https://example.com' },

```

### verify

Verify and decode a JWT.

```ts
function verify<T extends JWTPayload = JWTPayload>(
  token: string,
  secret: string | Uint8Array,
): Promise<T>;
```

Throws `InvalidTokenError` if the signature is invalid or the token is expired.

```ts title="middleware.ts"
import { verify } from '@holeauth/core/jwt';

try {
  const payload = await verify(token, process.env.JWT_SECRET!);
  console.log(payload.userId);
} catch (error) {
  // Invalid or expired token
}
```

### decode

Decode a JWT without verification (unsafe — use only for inspection).

```ts
function decode<T extends JWTPayload = JWTPayload>(token: string): T;
```

<Callout type="warn">
  This function does NOT verify the signature. Only use it for debugging or when the token was already verified upstream.
</Callout>

### rotateRefresh

Rotate a refresh token into a new access token and optionally a new refresh token.

```ts
function rotateRefresh(
  config: HoleauthConfig,
  refreshToken: string,
): Promise<{ accessToken: string; refreshToken: string; session: SessionData }>;
```

Throws `InvalidTokenError` if the refresh token is invalid or revoked. Implements reuse detection: if the same refresh token is used twice, the entire session family is revoked.

```ts title="app/api/auth/refresh/route.ts"
import { rotateRefresh } from '@holeauth/core/session';
import { auth } from '@/lib/auth';

try {
  const { accessToken, refreshToken, session } = await rotateRefresh(auth, oldRefreshToken);
  // Set new cookies
} catch (error) {
  // Revoke session family, clear cookies
}
```

### validateSession

Verify an access token and return the decoded session data.

```ts
function validateSession(
  config: HoleauthConfig,
  accessToken: string,
): Promise<SessionData | null>;
```

Returns `null` if the token is invalid or expired. Does not throw.

```ts title="middleware.ts"
import { validateSession } from '@holeauth/core/session';
import { auth } from '@/lib/auth';

const session = await validateSession(auth, accessToken);
if (!session) {
  return new Response('Unauthorized', { status: 401 });
}
```

## @holeauth/core/sso

SSO consumer: authorize and callback flows for Google, GitHub, Discord, Microsoft, and generic OIDC providers.

### authorize

Start the OAuth authorization flow by redirecting the user to the provider.

```ts
function authorize(
  providerId: string,
  config: SsoConfig,
): { redirectUrl: string; state: string; codeVerifier?: string };
```

```ts title="app/api/auth/authorize/[provider]/route.ts"
import { authorize } from '@holeauth/core/sso';
import { auth } from '@/lib/auth';

const { redirectUrl, state, codeVerifier } = authorize('google', auth.sso);
// Store state and codeVerifier in a cookie
// Redirect to redirectUrl
```

### callback

Handle the OAuth callback and exchange the authorization code for an access token.

```ts
function callback(
  providerId: string,
  input: { code: string; state: string; codeVerifier?: string },
  config: SsoConfig,
): Promise<{ providerAccountId: string; email: string; accessToken: string; refreshToken?: string }>;
```

```ts title="app/api/auth/callback/[provider]/route.ts"
import { callback } from '@holeauth/core/sso';
import { auth } from '@/lib/auth';

const result = await callback('google', {
  code: searchParams.get('code')!,
  state: searchParams.get('state')!,
  codeVerifier: cookies.get('holeauth.pkce'),
}, auth.sso);

// Find or create user with result.providerAccountId and result.email
```

### Providers

Built-in SSO consumer providers (no separate package required):

- `GoogleProvider({ clientId, clientSecret, redirectUri })` — Google OAuth 2.0
- `GithubProvider({ clientId, clientSecret, redirectUri })` — GitHub OAuth 2.0
- `DiscordProvider({ clientId, clientSecret, redirectUri })` — Discord OAuth 2.0
- `MicrosoftProvider({ clientId, clientSecret, redirectUri, tenant? })` — Microsoft OAuth 2.0
- `OIDCProvider({ clientId, clientSecret, redirectUri, issuer, scopes? })` — Generic OIDC

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { GoogleProvider, GithubProvider } from '@holeauth/core/sso';

export const auth = defineHoleauth({
  // ...
  sso: {
    baseUrl: process.env.NEXT_PUBLIC_BASE_URL!,
    providers: [
      GoogleProvider({
        clientId: process.env.GOOGLE_CLIENT_ID!,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
        redirectUri: `${process.env.NEXT_PUBLIC_BASE_URL}/api/auth/callback/google`,
      }),
      GithubProvider({
        clientId: process.env.GITHUB_CLIENT_ID!,
        clientSecret: process.env.GITHUB_CLIENT_SECRET!,
        redirectUri: `${process.env.NEXT_PUBLIC_BASE_URL}/api/auth/callback/github`,
      }),
    ],
  },
});
```

<Callout type="info">
  SSO consumer functionality is built into `@holeauth/core/sso`. No separate `plugin-idp-consumer` package is required. The framework adapters (Next.js, Express, Hono) automatically register the `/authorize/:provider` and `/callback/:provider` routes.
</Callout>

## Invites

Holeauth ships a complete invite flow on top of the `verificationToken` adapter. The flow issues a short-lived signed JWT, stores its hash for one-time consumption, and creates the user account when the invitee finishes registration. Invites continue to work even when `registration.selfServe` is `false`.

The methods below are exposed on the `HoleauthInstance` returned by `defineHoleauth`. Framework bindings route them under `/api/auth/invite/*` by default.

### getInviteInfo

Decode an invite token without consuming it. Useful for pre-filling a registration form.

```ts
function getInviteInfo(input: { token: string }): Promise<InviteClaims>;
```

Returns the embedded `email`, `name`, `groupIds`, `invitedBy`, `metadata`, `expiresAt`, and `identifier`. Throws `CredentialsError` (401) when the token is malformed, expired, or not an invite token.

```ts title="app/invite/[token]/page.tsx"
import { auth } from '@/lib/auth';

const claims = await auth.getInviteInfo({ token: params.token });
// claims.email, claims.name, claims.groupIds, ...
```

### listInvites

List all outstanding invite identifiers.

```ts
function listInvites(): Promise<InviteListEntry[]>;
```

Returns `{ identifier, email, expiresAt }` entries. Useful for an admin dashboard.

### revokeInvite

Invalidate an invite by its identifier.

```ts
function revokeInvite(input: { identifier: string }): Promise<void>;
```

The identifier is the `identifier` field returned from `createInvite` (also the JWT `jti`). After revocation, `consumeInvite` rejects with `CredentialsError`.

## @holeauth/core/errors

All thrown errors extend `HoleauthError`, which carries a stable `code` and an HTTP `status`. Framework bindings translate these directly into JSON responses.

```ts
import { HoleauthError, InvalidTokenError } from '@holeauth/core';

try {
  await auth.refresh({ refreshToken });
} catch (err) {
  if (err instanceof InvalidTokenError) {
    // 401 — clear cookies, redirect to /login
  }
}
```

| Class | Code | Status | Thrown when |
| --- | --- | --- | --- |
| `HoleauthError` | `*` | configurable | Base class; thrown directly for adapter-specific cases. |
| `InvalidTokenError` | `INVALID_TOKEN` | `401` | JWT signature invalid, token malformed, or refresh token unknown. |
| `SessionExpiredError` | `SESSION_EXPIRED` | `401` | Access token is past its `exp`. |
| `RefreshReuseError` | `REFRESH_REUSE` | `401` | A refresh token was presented after it was already rotated — the entire session family is revoked. |
| `CredentialsError` | `INVALID_CREDENTIALS` | `401` | Password mismatch, invite consumed, or pending challenge token invalid. |
| `CsrfError` | `CSRF_FAILED` | `403` | Double-submit cookie mismatch on a mutating request. |
| `RegistrationDisabledError` | `REGISTRATION_DISABLED` | `403` | `register` flow called while `registration.selfServe === false`. |
| `AccountConflictError` | `ACCOUNT_CONFLICT` | `409` | Email already registered (sign-up, invite, or SSO link). |
| `PendingChallengeError` | `PENDING_CHALLENGE` | `401` | A plugin halted sign-in and requires follow-up verification (2FA, etc.). |
| `ProviderError` | `PROVIDER_ERROR` | `502` | Upstream SSO provider responded with an error. |
| `AdapterError` | `ADAPTER_ERROR` | `500` | Adapter implementation raised an unexpected error. |
| `NotSupportedError` | `NOT_SUPPORTED` | `501` | Adapter is missing a required optional method. |

<Callout type="info">
  Framework bindings catch every `HoleauthError` and serialise it as `{ error: { code, message } }` with the matching HTTP status. You normally do not need to handle these manually unless you call core functions directly.
</Callout>
