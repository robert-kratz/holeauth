Provides a drop-in catch-all route handler, a middleware helper, and cookie utilities.

## `createAuthHandler(config, opts?)`

Wraps `defineHoleauth` and adds Next.js-aware behavior:

```ts
import { createAuthHandler } from '@holeauth/nextjs';

export const auth = createAuthHandler(
  config,
  {
    basePath: '/api/auth',       // default
    defaultRedirect: '/dashboard', // post-SSO-callback redirect
  },
);
```

The returned object extends `HoleauthInstance` with:

```ts
interface NextHoleauth extends HoleauthInstance {
  handlers: {
    GET: (req: Request) => Promise<Response>;
    POST: (req: Request) => Promise<Response>;
  };
  // Reads next/headers when no token is passed
  getSession(accessToken?: string): Promise<SessionData | null>;
}
```

Mount it once:

```ts
// app/api/auth/[...holeauth]/route.ts
import { auth } from '@/lib/auth';

export const runtime = 'nodejs'; // argon2 & @simplewebauthn/server are Node-only

export const { GET, POST } = auth.handlers;
```

All endpoints are listed in the [API reference](/docs/api/endpoints).

## `holeauthMiddleware(opts)`

A middleware helper that redirects unauthenticated / pending-2FA users:

```ts
// middleware.ts
import { holeauthMiddleware } from '@holeauth/nextjs';
import { auth } from '@/lib/auth';

export const middleware = holeauthMiddleware({
  config: auth.config,
  protect: ['/dashboard', /^\/account\/.*/],
  signInPath: '/login',        // default '/login'
  twoFactorPath: '/2fa/verify', // default '/2fa/verify'
});

export const config = {
  matcher: ['/((?!_next|public|api/auth).*)'],
};
```

Behavior per request:

- Has a valid access cookie â†’ continue.
- Has a `pending2fa` cookie â†’ redirect to `twoFactorPath`.
- Path matches `protect` but no auth cookie â†’ redirect to `signInPath`.
- Otherwise â†’ continue.

> Middleware runs on the **Edge** runtime. Only validates JWTs â€” never hashes passwords or calls adapters.

## Cookie helpers

Useful when you build custom routes:

```ts
import {
  writeAuthCookies,
  clearAuthCookies,
  writePending2fa,
  readCookie,
  checkCsrf,
} from '@holeauth/nextjs';

// In a Route Handler
export async function POST(req: Request) {
  if (!checkCsrf(req, auth.config)) {
    return new Response('Forbidden', { status: 403 });
  }
  const res = new Response(null);
  writeAuthCookies(auth.config, res.headers, tokens);
  return res;
}
```

## Server-component session reads

`auth.getSession()` with no argument reads the `holeauth.access` cookie from `next/headers()`. Use it anywhere you'd normally call `cookies()`:

```tsx
const session = await auth.getSession();
```

Pass an explicit token if you're reading it from somewhere else (for example a server-to-server request):

```ts
const session = await auth.getSession(req.headers.get('authorization')?.slice(7));
```

## `validateCurrentRequest(auth, opts?)`

One-stop guard for Server Components and Route Handlers. Reads the session cookie, optionally loads the full user row and RBAC snapshot, and redirects (or throws) on failure.

```tsx title="app/dashboard/page.tsx"
import { validateCurrentRequest } from '@holeauth/nextjs';
import { auth } from '@/lib/auth';

export default async function DashboardPage() {
  const { session, user } = await validateCurrentRequest(auth, {
    redirectTo: '/login',
    loadUser: true,
    permissions: 'dashboard.view',
  });

  return <h1>Welcome, {user?.email}</h1>;
}
```

## `getFullSession(auth, opts?)`

Non-redirecting variant of `validateCurrentRequest`. Returns `null` on failure instead of calling `redirect()`.

```tsx title="app/layout.tsx"
import { getFullSession } from '@holeauth/nextjs';
import { auth } from '@/lib/auth';

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const validated = await getFullSession(auth, { loadUser: true });

  return (
    <html>
      <body>
        <AuthenticatedProvider value={validated}>
          {children}
        </AuthenticatedProvider>
      </body>
    </html>
  );
}
```

Accepts the same options as `validateCurrentRequest` minus `redirectTo` and `throwOnFail`.

**Returns:** `Promise<ValidatedRequest | null>`

## `getSessionOrRefresh(req, instance)`

Read access + refresh cookies from a `Request`, validate the JWT, and silently rotate the refresh token if needed. Returns `Set-Cookie` strings the caller must forward on the outgoing response.

Designed for server middleware, tRPC context factories, and custom route handlers that need transparent session refresh without framework magic.

```ts title="trpc/context.ts"
import { getSessionOrRefresh } from '@holeauth/nextjs';
import { auth } from '@/lib/auth';

export async function createContext({ req, resHeaders }: FetchCreateContextFnOptions) {
  const { session, setCookieHeaders } = await getSessionOrRefresh(req, auth);
  for (const c of setCookieHeaders) resHeaders.append('Set-Cookie', c);
  return { session };
}
```

**Returns:**

## `createDispatcher(instance, opts?)`

Low-level factory that builds the `GET`/`POST` handler function that `createAuthHandler` uses internally. Useful when you need to mount the handler somewhere other than the standard catch-all route.

```ts
import { createDispatcher } from '@holeauth/nextjs';

const dispatch = createDispatcher(authInstance, {
  basePath: '/api/auth',
});

export const GET = dispatch;
export const POST = dispatch;
```

## Cookie helpers

All helpers are exported from `@holeauth/nextjs` and operate on `Headers` objects.

### `writeAuthCookies(config, headers, tokens)`

Append the three auth cookies (access, refresh, csrf) to a `Headers` object.

```ts
import { writeAuthCookies } from '@holeauth/nextjs';

const res = new Response(null, { status: 204 });
writeAuthCookies(auth.config, res.headers, tokens);
return res;
```

### `clearAuthCookies(config, headers)`

Expire all auth cookies (access, refresh, csrf, pending).

```ts
import { clearAuthCookies } from '@holeauth/nextjs';

clearAuthCookies(auth.config, res.headers);
```

### `writePending(config, headers, pendingToken)`

Set the `pending` cookie used during 2FA / passkey challenge steps.

```ts
import { writePending } from '@holeauth/nextjs';

writePending(auth.config, res.headers, pendingToken);
```

### `readCookie(req, config, kind)`

Read a single holeauth cookie from a `Request`.

```ts
import { readCookie } from '@holeauth/nextjs';

const access = readCookie(req, auth.config, 'access');
```

### `checkCsrf(req, config)`

Return `true` if the double-submit CSRF check passes (`x-holeauth-csrf` header matches the `csrf` cookie).

```ts
import { checkCsrf } from '@holeauth/nextjs';

if (!checkCsrf(req, auth.config)) {
  return new Response('Forbidden', { status: 403 });
}
```

### `parseCookies(header)`

Parse a raw `Cookie` header string into a `Record<string, string>`.

```ts
import { parseCookies } from '@holeauth/nextjs';

const jar = parseCookies(req.headers.get('cookie'));
```
