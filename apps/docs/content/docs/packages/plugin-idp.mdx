# @holeauth/plugin-idp

OpenID Connect + OAuth 2.0 Identity Provider plugin for holeauth. Turns your holeauth instance into a first-party OIDC provider that other apps can authenticate against.

**Implements:**
- Authorization Code flow with PKCE (public and confidential clients)
- Refresh token rotation with family-revoke on reuse
- OIDC Discovery (`.well-known/openid-configuration`) + JWKS + userinfo
- Token revocation (RFC 7009)
- RP-initiated logout (`/oauth2/end-session`)
- Team-based app ownership with owner/developer roles
- Remembered consent (per user × app × scope set)

<Callout type="info">
  Access tokens are JWT (RS256 by default) signed with IdP-owned keys — **not** the holeauth core session secret. Refresh tokens are opaque and stored as SHA-256 hashes.
</Callout>

## Quick example

```ts title="lib/auth.ts"
import { defineHoleauth } from '@holeauth/core';
import { idp } from '@holeauth/plugin-idp';
import { createIdpAdapter } from '@holeauth/idp-drizzle/pg';
import { db } from '@/db';

const idpAdapter = createIdpAdapter({ db });

export const auth = defineHoleauth({
  // ...adapters and secrets
  plugins: [
    idp({
      adapter: idpAdapter,
      issuer: 'https://auth.example.com',
    }),
  ],
});
```

After bootstrapping, call `auth.plugins.idp.keys.bootstrap()` once to generate the initial signing key.

## `idp(options)`

## `IdpApi` methods

Accessible as `auth.plugins.idp.*` after setup.

### `meta`

Static metadata used to build the OIDC discovery document.

```ts
const { issuer, scopesSupported } = auth.plugins.idp.meta;
```

### `apps.*`

#### `apps.create(callerUserId, input)`

Create a new OAuth app. For confidential apps, the `clientSecret` is returned **once** — it is hashed before storage.

```ts
const { app, clientSecret } = await auth.plugins.idp.apps.create(userId, {
  name: 'My App',
  type: 'confidential',
  redirectUris: ['https://myapp.com/callback'],
  teamId: teamId,
});
// Show clientSecret to the user NOW — it cannot be retrieved later
```

<TypeTable
  type={{
    name: { description: 'App display name.', type: 'string', required: true },
    description: { description: 'Optional description.', type: 'string | null' },
    type: { description: '"public" for SPAs/native apps, "confidential" for server-side apps.', type: '"public" | "confidential"', required: true },
    redirectUris: { description: 'Allowed OAuth redirect URIs.', type: 'string[]', required: true },
    allowedScopes: { description: 'Subset of issuer scopes this app may request.', type: 'string[]' },
    requirePkce: { description: 'Force PKCE for all token exchanges.', type: 'boolean' },
    teamId: { description: 'Team to own the app. If omitted, a new team is created for the caller.', type: 'string' },
  }}
/>

#### `apps.listForUser(userId)`

List all apps the user can access via their team memberships.

#### `apps.listAll()`

Admin: list all apps across all teams.

#### `apps.get(callerUserId, appId, opts?)`

Fetch a single app. Caller must be a team member unless `opts.admin` is true.

#### `apps.update(callerUserId, appId, patch, opts?)`

Update app metadata (name, description, logo, redirect URIs, allowed scopes, requirePkce, disabled).

#### `apps.regenerateSecret(callerUserId, appId)`

Generate a new `clientSecret` for a confidential app. Returns it once in plaintext.

#### `apps.delete(callerUserId, appId, opts?)`

Delete an app and revoke all its refresh tokens.

### `teams.*`

#### `teams.create(ownerUserId, name)`

Create a new team. The caller is automatically assigned the `owner` role.

#### `teams.listForUser(userId)`

Return all teams the user belongs to, with their role.

```ts
const teams = await auth.plugins.idp.teams.listForUser(userId);
// Array<IdpTeam & { role: 'owner' | 'developer' }>
```

#### `teams.listMembers(callerUserId, teamId)`

List all members of a team.

#### `teams.addMember(callerUserId, teamId, userId, role)`

Add a member to a team. Caller must be an owner.

#### `teams.removeMember(callerUserId, teamId, userId)`

Remove a member. Caller must be an owner.

### `tokens.*`

#### `tokens.listForApp(callerUserId, appId)`

List all active refresh tokens for an app (admin operation).

#### `tokens.revokeAllForApp(callerUserId, appId)`

Revoke all refresh tokens for an app (e.g. when rotating the client secret).

### `keys.*`

#### `keys.bootstrap()`

Generate the initial RS256 signing key. Call once at application startup.

```ts
await auth.plugins.idp.keys.bootstrap();
```

#### `keys.rotate()`

Generate a new signing key and mark the previous one as rotated. Old keys remain in JWKS until no tokens signed by them remain valid.

```ts
const newKey = await auth.plugins.idp.keys.rotate();
```

## Standalone utilities

### `rotateSigningKey(adapter, alg?)`

Low-level key rotation, called internally by `keys.rotate()`.

```ts
import { rotateSigningKey } from '@holeauth/plugin-idp';
```

### `ensureSigningKey(adapter, alg?)`

Ensure at least one active signing key exists; create one if not. Used internally by `keys.bootstrap()`.

```ts
import { ensureSigningKey } from '@holeauth/plugin-idp';
```

### `renderConsentPage(input)`

Render the built-in HTML consent page. Override by implementing your own `/oauth2/authorize` handler.

```ts
import { renderConsentPage } from '@holeauth/plugin-idp';

const html = renderConsentPage({
  appName: app.name,
  scopes: ['openid', 'email'],
  cancelUrl: '/oauth2/cancel',
  submitUrl: '/oauth2/authorize',
});
return new Response(html, { headers: { 'content-type': 'text/html' } });
```

## OIDC Routes

The plugin mounts these routes under the `basePath`:

| Method | Path | Notes |
|--------|------|-------|
| GET | `/.well-known/openid-configuration` | Discovery document |
| GET | `/.well-known/jwks.json` | Public keys |
| GET | `/oauth2/authorize` | Authorization endpoint |
| POST | `/oauth2/token` | Token exchange |
| POST | `/oauth2/revoke` | Token revocation (RFC 7009) |
| GET | `/oauth2/userinfo` | Userinfo endpoint |
| GET | `/oauth2/end-session` | RP-initiated logout |
